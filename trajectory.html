軌跡分析 (Trajectory / Pseudotime)

從靜態快照推估細胞狀態轉變的「時間軸」——從幹細胞、發育、疾病進程到 RNA velocity 的動態方向。

Infer the "timeline" of cell state transitions from static snapshots — from stemness and development to disease progression and the directional cues of RNA velocity.

概覽

一、為什麼從快照能推時間？

scRNA-seq 是單一時間點的快照，理論上「沒有時間」。但生物系統的關鍵直覺是：在同一個時間點，組織中存在處於分化、活化、衰老等不同階段的細胞。如果這些細胞之間有連續的表達變化，我們就能依「相鄰細胞表達相似」的邏輯，把細胞排成一條擬時序 (pseudotime) 軸，重建「從新生到終端」的動態過程。

常見應用：幹細胞 → 終端細胞的分化路徑、naive → effector → memory T cell 的活化、疾病早期 → 末期的進展、細胞週期 (G1 → S → G2 → M)、發育階段重建。

scRNA-seq is a single-timepoint snapshot — in principle, "no time". But the biological intuition is key: at any moment, a tissue contains cells at different stages of differentiation, activation, senescence. If their expression varies continuously, we can — under the assumption "neighboring cells share expression" — order them along a pseudotime axis to reconstruct the dynamics from naïve to terminal.

Common uses: stem → terminal differentiation, naive → effector → memory T-cell activation, early → late disease progression, cell cycle (G1 → S → G2 → M), developmental staging.

🌱

發育分化

造血幹細胞 → 分化譜系、神經前驅 → 終端神經元；軌跡可揭示分支點 (fate decision)。

HSCs → blood lineages, neural progenitors → mature neurons; trajectories expose fate-decision branch points.

⚕️

疾病進程

例如腫瘤亞群從上皮 → EMT → 侵襲性；纖維化從靜止 fibroblast → 活化 myofibroblast。

Tumor subpopulations: epithelial → EMT → invasive; fibrosis: quiescent fibroblast → activated myofibroblast.

🔄

細胞週期

G1 → S → G2 → M 的周期性表達是常見的「假軌跡」——分析時必須先回歸掉或排除。

The cyclic G1 → S → G2 → M signature is a frequent spurious trajectory — regress it out or exclude before analysis.

💡
核心前提：軌跡分析需要數據中存在「中間狀態」的細胞。如果樣本只包含兩個完全分化的終端群體，演算法仍會強行畫出一條軌跡，但這條軌跡沒有生物學意義——它只是兩團點之間最短距離的連線。先用 marker gene 與分群結果判斷分化過程是否存在，再決定要不要做 pseudotime。 Core prerequisite: trajectory analysis requires intermediate-state cells. If your sample contains only two fully differentiated populations, the algorithm will still draw a trajectory — but it carries no biological meaning; it's just the shortest path between two blobs. Verify intermediate states via markers and clustering first, then decide whether pseudotime is even applicable.
方法分類

二、核心方法的家族樹

軌跡分析數十種工具，可大致分為三大家族：

Dozens of trajectory tools fall broadly into three families:

🌳 家族 1：樹/曲線型

  • Monocle 2 (DDRTree)：在 reduced 空間學一棵 reversed-graph embedding tree。
  • Monocle 3：以 UMAP 為底，再用 principal graph 學習拓撲；支援 disjoint trajectories、branching、cycle。
  • Slingshot：在已分群的 reduced 空間擬合 principal curve，先 MST 連接 cluster 再用 simultaneous principal curves 平滑每條 lineage。
  • Monocle 2 (DDRTree): learns a reversed-graph embedding tree in reduced space.
  • Monocle 3: UMAP-based; principal graph learning supports disjoint trajectories, branching, and cycles.
  • Slingshot: fits principal curves on clustered, reduced data — MST over clusters, then simultaneous principal curves per lineage.

🕸️ 家族 2：圖型

  • PAGA (Partition-based Graph Abstraction)：先做 cluster，再用統計檢定決定 cluster 間是否「連通」。不強制一條軌跡，輸出一張 cluster 拓撲圖——適合大尺度概覽。
  • PAGA → init：PAGA 結果可作為其他工具（如 ForceAtlas2、UMAP）的初始化，產出可解釋的全景。
  • PAGA: clusters first, then statistical tests decide which clusters are connected. Does not force a single trajectory — outputs a topology graph, ideal for large-scale overview.
  • PAGA → init: PAGA can initialize ForceAtlas2/UMAP for interpretable global layouts.

🧭 家族 3：方向性

  • RNA Velocity (La Manno et al., Nature 2018, velocyto)：利用 spliced / unspliced mRNA 比率推估每個細胞「正在朝哪走」。
  • scVelo：將 velocity 升級為 dynamical model，估計每基因的 splicing kinetics。
  • CellRank：把 pseudotime、velocity、實驗時間整合進 Markov chain，輸出 macrostate（terminal / initial / intermediate）與 fate probability。
  • RNA Velocity (La Manno et al., Nature 2018, velocyto): spliced/unspliced ratio infers each cell's instantaneous direction.
  • scVelo: upgrades velocity to a dynamical model with per-gene splicing kinetics.
  • CellRank: unifies pseudotime + velocity + experimental time into a Markov chain; returns macrostates (terminal/initial/intermediate) and fate probabilities.

深入 RNA velocity

四、RNA Velocity 的核心邏輯

RNA velocity 由 La Manno et al. (Nature, 2018) 引入：scRNA-seq 的 short-read 比對中，含 intron 的 reads 來自未剪接 (unspliced, nascent) mRNA，純 exon 的 reads 來自已剪接 (spliced, mature) mRNA。因為轉錄一個基因時，nascent 必須先存在、然後被剪接，所以：

d(spliced)/dt = β · unspliced − γ · spliced

如果某個基因目前 unspliced 比例「異常高」(超過 steady-state)，代表它正在被啟動轉錄；反之則代表正在被關閉。把所有基因的瞬時導數合成一個向量場 → 細胞「正在朝哪邊轉變」。

兩種模型

  • Steady-state (stochastic)：假設每個基因在 high-expression 與 low-expression 細胞之間達 steady state；用 quantile 線性回歸估 γ。簡單、快，但對 transient 過程不夠精確。
  • Dynamical model (scVelo)：對每個基因估計 transcription α、splicing β、degradation γ 動力學參數，能解析 induction/repression 階段。較準確但需更大數據量。

Introduced by La Manno et al. (Nature 2018): in scRNA-seq read alignment, intron-containing reads originate from unspliced (nascent) mRNA, and exon-only reads from spliced (mature) mRNA. Because nascent precedes mature in time:

d(spliced)/dt = β · unspliced − γ · spliced

If a gene's unspliced fraction is "above steady state", it is being induced; below means repression. Aggregating per-gene derivatives gives a velocity vector field → where each cell is heading next.

Two flavors

  • Steady-state (stochastic): assumes each gene reaches steady state in high/low expression cells; γ from quantile regression. Fast but inaccurate for transients.
  • Dynamical model (scVelo): fits per-gene transcription α, splicing β, degradation γ — resolves induction/repression phases. More accurate, more data-hungry.
⚠️
velocity 不是萬靈丹：需要可靠的 unspliced 量化。10x 3' chemistry 的 unspliced 訊號相對稀薄；在 已分化、低週轉率 組織（成年神經元、心肌）尤其不可靠，velocity 方向可能反向或變得隨機。新方法 UniTVelo、VeloVAE、veloVI 嘗試改善這些問題。先用 scv.pl.proportions() 檢查 unspliced 比例是否合理（通常 10–25%）。 Velocity is not a panacea: it depends on reliable unspliced quantification. 10x 3' chemistry yields sparse unspliced reads; in terminally differentiated, low-turnover tissues (adult neurons, cardiomyocytes), velocity can be unreliable or even reversed. Newer methods (UniTVelo, VeloVAE, veloVI) tackle this. Always check the unspliced proportion (usually 10–25%) with scv.pl.proportions() first.

驗證 trajectory

七、如何判斷軌跡是「真的」？

任何 pseudotime 演算法都會「給你答案」，但答案的可信度需另外驗證。下列三項是業界最常用的健全性檢查：

Every pseudotime tool will hand you an answer — its credibility needs separate verification. Three industry-standard sanity checks:

📈 ① Marker 單調變化

已知的幹細胞 marker（如 CD34、SOX2）應沿 pseudotime 單調下降；終端 marker（如 HBB 對紅血球、MS4A1 對 mature B）應單調上升。若曲線雜亂或反向，軌跡可能錯誤或方向反了。

Known stem markers (CD34, SOX2) should decrease monotonically with pseudotime; terminal markers (HBB for erythrocytes, MS4A1 for mature B) should rise monotonically. If the curve is messy or inverted, your trajectory is wrong or reversed.

🧪 ② 重現 landmark

例：B cell 成熟順序應為 pro-B → pre-B → immature → mature。若你的軌跡把 immature 排在 mature 後面，立即懷疑。

e.g., B-cell maturation should follow pro-B → pre-B → immature → mature. If your trajectory places immature after mature, suspect immediately.

🧭 ③ velocity 交叉驗證

若同時跑 pseudotime 與 RNA velocity，兩者方向應一致。不一致時通常是 root cell 選錯或某條 lineage 的 velocity 不可靠。

If you run both pseudotime and velocity, their directions should agree. Disagreements usually trace to a wrong root or unreliable velocity for one lineage.

最佳實務

八、最佳實務與常見陷阱

✅ 最佳實務清單

①
先 cluster、再 trajectory。把 trajectory 強加在 noise 上沒有意義。先確認有合理的 cluster 結構與中間態。
②
UMAP 只用來看，不用來算。UMAP 適合視覺化但會扭曲距離。請在 PCA / harmony 整合空間重新計算 k-NN graph 再丟給 trajectory 工具。
③
先 PAGA 看全景，再 Monocle/Slingshot 細節。PAGA 告訴你「哪些 cluster 連通」，能避免錯把無關 cluster 連成一條 lineage。
④
謹慎選 root cell。用生物學先驗（幹細胞 marker、CytoTRACE diversity score、或 velocity 起點）決定，避免隨機點選。
⑤
細胞週期務必處理。否則 G1 → S → G2 → M 的周期性會蓋過你想看的分化訊號。Seurat CellCycleScoring + regress out 或排除 cycling cells。
①
Cluster first, then trajectory. Forcing a trajectory onto noise is meaningless — verify cluster structure and intermediate states first.
②
Use UMAP for viz only, not computation. UMAP is great visually but distorts distances. Recompute the k-NN graph in PCA / harmony space before feeding trajectory tools.
③
PAGA for global, then Monocle/Slingshot for detail. PAGA tells you which clusters are connected, preventing spurious lineages between unrelated clusters.
④
Pick the root cell carefully. Use biological priors (stem markers, CytoTRACE diversity, or velocity origin) — never random.
⑤
Address cell cycle. Otherwise the G1→S→G2→M cycle dominates the signal you care about. Use Seurat CellCycleScoring + regress out, or exclude cycling cells.

🚧 常見陷阱 Top 5

  1. UMAP 的視覺 lineage 騙人：UMAP 上看起來連續的兩團 cluster，背後可能根本沒有真實連通——務必用 PAGA 統計檢定或 velocity 確認。
  2. Root cell 選錯：起點不同，整條軌跡的方向、分支點全跟著錯。Trajectory 是「對 root 的相對距離」，root 錯，全錯。
  3. 把細胞週期當分化：cycling 細胞會在 UMAP 上沿 G1→S→G2→M 排成一條漂亮的弧——很容易被誤認為分化軌跡。
  4. 忽略增殖：許多 lineage 在分化時細胞數會擴張（progenitor → 大量 mature 細胞）。Pseudotime 排序不含「該位置的細胞數權重」，可能讓 progenitor 看起來「比實際少」。
  5. 把 velocity 用在錯的組織：成年穩定組織（神經元、心肌）unspliced 訊號很弱，velocity 方向不可信，甚至會出現整體反向。先檢查 unspliced 比例與 latent time 一致性。
  1. UMAP visual "lineage" lies: clusters that look continuous on UMAP may not be truly connected — verify with PAGA tests or velocity.
  2. Wrong root cell: changing the root flips direction and reshapes branch points. Pseudotime is "distance from root" — wrong root, wrong everything.
  3. Mistaking cell cycle for differentiation: cycling cells trace a beautiful G1→S→G2→M arc on UMAP — easily misread as a differentiation trajectory.
  4. Ignoring proliferation: many lineages expand cell numbers during differentiation (progenitor → many mature cells). Pseudotime ordering carries no "cell-count weighting", so progenitors look "rarer than they really are".
  5. Velocity in wrong tissue: adult quiescent tissues (neurons, cardiomyocytes) yield sparse unspliced — velocity is unreliable and may even invert globally. Check unspliced proportion and latent-time consistency first.
