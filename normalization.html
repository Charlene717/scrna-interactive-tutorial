標準化 (Normalization)

消除技術偏差，揭示真實的生物學差異。探索從傳統 LogNormalize 到現代 SCTransform 的演進。

Remove technical biases to reveal true biological differences. Explore the evolution from LogNormalize to SCTransform.

概覽

一、為什麼需要標準化？

在 scRNA-seq 中，即使是同一個細胞，如果分別重新捕獲、建庫、定序兩次，得到的 UMI 計數也會明顯不同。這些技術性差異混雜在真實生物訊號裡，必須先校正掉捕獲深度，才能比較細胞間的「相對表達」。

更棘手的是，scRNA-seq 數據有三個獨特特性：(1) 極度稀疏——多數基因對多數細胞讀數為 0；(2) 計數型分佈——非連續、右偏、過離散 (overdispersed) 的負二項分佈；(3) 異質方差——高表達基因的方差遠大於低表達基因。傳統 bulk RNA-seq 的 TPM/CPM 並不能直接套用，這也是 SCTransform 等專屬方法誕生的原因。

In scRNA-seq, even the same cell, if re-captured/library-prepped/sequenced twice, will yield noticeably different UMI counts. These technical differences are entangled with real biological signal, so capture depth must be corrected before we can compare "relative expression" across cells.

What makes it harder: scRNA-seq data has three unique features — (1) extreme sparsity (most genes are zero in most cells); (2) count distribution — discrete, right-skewed, overdispersed negative-binomial; (3) heteroskedasticity — high-expression genes have far larger variance than low ones. Bulk RNA-seq's TPM/CPM doesn't transfer, which is why dedicated methods like SCTransform were developed.

📊

定序深度

不同細胞在測序儀上獲得的總 Reads 數不同——一個細胞可能 5000 reads，另一個 50000 reads。這是純粹的技術偏差。

Different cells receive different total reads — one cell might get 5,000 reads while another gets 50,000. Pure technical bias.

🧲

捕獲效率

逆轉錄、PCR 擴增、二代定序各步驟都有隨機損失。實際 mRNA 分子被偵測到的比例僅 10–30%。

Reverse transcription, PCR amplification and sequencing each lose molecules randomly. Only 10–30% of original mRNAs are detected.

💧

零膨脹

單細胞數據中 70–95% 的數值是 0。許多 0 是「技術性 dropout」（基因實際表達但未被捕獲），少數是真正不表達。LogNormalize 不區分兩者。

70–95% of values in a single-cell matrix are zero. Many are "technical dropouts" (gene is expressed but missed); few are true zeros. LogNormalize cannot distinguish them.

🧬

RNA 總量差異

靜止淋巴細胞 vs. 增殖中的腫瘤細胞，本身 mRNA 總量就差數倍。這既是技術問題也是生物學議題——「等比例縮放」會把大細胞的真實高表達削弱。

Quiescent lymphocytes vs. proliferating tumor cells differ in total mRNA by several-fold. This is both technical and biological — proportional scaling can mask true high expression in large cells.

💡
記住：Normalization 校正的是捕獲深度，不是批次效應。批次差異（不同樣本、不同 lane、不同建庫日期）需要 Integration / Harmony / scVI 等專門方法處理。 Remember: Normalization corrects capture depth, not batch effects. Batch differences (different samples/lanes/library days) require dedicated integration methods like Harmony or scVI.
原理

二、三大主流方法的數學原理

① LogNormalize (Seurat default · Scanpy normalize_total+log1p)

公式： x'ij = log( xij / Σj xij × 10000 + 1 )

對細胞 j 的每個基因 i 的原始計數 xij，先除以該細胞的總計數（library size），乘以 scale factor（通常 10000），最後 log(x+1) 壓縮動態範圍。輸出存於 data slot。

優點：計算極快、概念直觀、與 bulk CPM 一脈相承。
缺點：(a) 全細胞共用同一 size factor 估計（總 UMI），對極端細胞不準；(b) log 後仍有異質方差——高表達基因方差較大；(c) 假設所有細胞 mRNA 總量相同，違反生物學事實。

Formula: x'ij = log( xij / Σj xij × 10000 + 1 )

For each gene i in cell j, divide by the cell's total counts (library size), multiply by a scale factor (usually 10000), then log(x+1) to compress dynamic range. Output stored in the data slot.

Pros: Extremely fast, intuitive, continuous with bulk CPM logic.
Cons: (a) Same size factor for all cells (total UMI) — inaccurate for extreme cells; (b) Residual heteroskedasticity after log — high-expression genes still have larger variance; (c) Assumes equal total mRNA across cells, violating biology.

② Scran pooling deconvolution (Lun et al. 2016)

核心想法：單個細胞的 UMI 太稀疏無法穩定估計 size factor。Scran 先把相似細胞合併成 pool（如 20–100 個一組）做穩健估計，再用線性反卷積（deconvolution）把 pool size factor 拆回到每個細胞。

數學流程：對細胞 i ∈ S（pool S）的計數總和 tS = Σi∈S xij / xref,j，建立線性方程組求解每個 θi（cell-specific size factor）。實際使用會先 quickCluster() 把細胞依表達相似度分群，避免跨型別混 pool 造成偏差。

優點：對 RNA 總量差異大的數據集（如肝臟+免疫混合）顯著優於簡單除以 total UMI。
缺點：對極稀疏的 droplet 數據（如平均 < 500 UMI/cell）可能不穩。

Idea: A single cell's UMI vector is too sparse to estimate a stable size factor. Scran pools similar cells (e.g., groups of 20–100) for a robust estimate, then uses linear deconvolution to recover each cell's size factor.

Procedure: For cells i in pool S, sum tS = Σi∈S xij / xref,j and solve a linear system for θi (cell-specific size factor). In practice, run quickCluster() first to group similar cells and avoid mixing cell types into the same pool.

Pros: Far better than dividing-by-total-UMI when RNA content varies a lot (e.g., hepatocytes + immune).
Cons: Can be unstable on very sparse droplet data (mean < 500 UMI/cell).

③ SCTransform (Hafemeister & Satija 2019 · Choudhary & Satija 2022 v2)

核心想法：對每個基因擬合一個負二項 (NB) 廣義線性模型，以 log(total UMI) 為唯一解釋變數，再對所有基因的擬合參數（β0, β1, θ）做核迴歸正則化 (kernel regularization)——同表達區間基因共享平滑後的參數，避免過擬合與稀有基因的不穩定估計。

輸出：Pearson residuals — rij = (xij − μ̂ij) / √(μ̂ij + μ̂ij²/θ̂)。直觀理解：每個 (gene, cell) 殘差代表「該細胞表達超出深度可預測的多少標準差」，理論上應為均值 0、方差 1 的均質化分佈，非常適合直接送 PCA。

v2 改進 (2022)：使用 glmGamPoi 加速 NB 擬合（快約 10×）、預設用 vst.flavor='v2'、更穩健的離散參數估計。
優點：同時處理深度校正 + 異質方差 + 過離散。
缺點：計算耗時（v2 後緩解）、產出 SCT assay 與原 RNA assay 並存容易混淆 downstream slot 操作。

Idea: Fit a negative binomial (NB) GLM for each gene with log(total UMI) as the sole covariate, then apply kernel regularization to the fitted parameters (β0, β1, θ) — genes with similar expression share smoothed parameters, avoiding overfit on rare genes.

Output: Pearson residuals — rij = (xij − μ̂ij) / √(μ̂ij + μ̂ij²/θ̂). Intuitively each (gene, cell) residual is "how many SDs the cell's expression exceeds what depth alone predicts". Theoretically mean=0, variance=1 — homoskedastic and perfect for PCA.

v2 (2022): Uses glmGamPoi to accelerate NB fitting (~10×); default vst.flavor='v2'; more robust dispersion estimation.
Pros: Handles depth + heteroskedasticity + overdispersion in one step.
Cons: Slower than LogNorm (alleviated in v2); SCT assay coexists with original RNA assay — easy to confuse downstream slot operations.

程式碼

五、實作範例

# ===== Seurat: 兩條主流路線 =====
library(Seurat)

# A. 傳統路線：LogNormalize → FindVariableFeatures → ScaleData
pbmc <- NormalizeData(pbmc,
  normalization.method = "LogNormalize",
  scale.factor = 10000)
# 預設僅做 log1p(x / total_UMI × 10000)；scale.factor 通常無需改

# B. 推薦路線：SCTransform v2 (一次完成 norm + HVG + scale)
pbmc <- SCTransform(pbmc,
  vst.flavor = "v2",
  vars.to.regress = "percent.mt",
  verbose = FALSE)

# ⚠ 之後 PCA / Clustering 都會自動使用 SCT assay
DefaultAssay(pbmc) # 確認當前 assay
pbmc <- RunPCA(pbmc) # 自動讀 SCT@scale.data

# 差異表現分析建議切回 RNA assay (LogNormalize-based)
DefaultAssay(pbmc) <- "RNA"
pbmc <- NormalizeData(pbmc) # DE 用 log-norm 值
import scanpy as sc

# ===== Scanpy: 兩條主流路線 =====

# A. 傳統 LogNormalize
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
adata.raw = adata.copy()  # 保存 log-norm 副本 (DE 用)

# B. Analytic Pearson residuals (NB-residual, SCT 等效)
sc.experimental.pp.normalize_pearson_residuals(adata)
# 殘差存於 adata.X；可後接 sc.pp.pca() 直接降維

# 也可用單一步驟完成 norm + HVG + PCA
sc.experimental.pp.recipe_pearson_residuals(
    adata, n_top_genes=2000)
# ===== scran: pool deconvolution 流程 =====
library(scran); library(scater)

sce <- as.SingleCellExperiment(pbmc)

# Step 1: quickCluster 把相似細胞分群
clusters <- quickCluster(sce, min.size = 100)

# Step 2: 計算 cell-specific size factor
sce <- computeSumFactors(sce,
  clusters = clusters, min.mean = 0.1)

# Step 3: log-normalize 套用 size factor
sce <- logNormCounts(sce)
# 結果存於 assay(sce, "logcounts")

# ⚠ 確認 size factor 分佈，若有負值代表 pool 太小要調整
summary(sizeFactors(sce))
最佳實務

六、最佳實務

📏 依規模選方法

細胞 < 5k：LogNormalize 即可；5k–100k：SCTransform v2；> 100k：考慮 Pearson residuals 或 scVI（GPU 加速 + 內建批次校正）。

< 5k cells: LogNormalize is fine. 5k–100k: SCTransform v2. > 100k: consider Pearson residuals or scVI (GPU + built-in batch correction).

🧯 回歸變數慎用

SCTransform 的 vars.to.regress 常見回歸 percent.mt；不要回歸 nCount_RNA（已被 NB 模型吸收）；研究細胞週期時不要回歸 S/G2M.Score。

Common: regress percent.mt in SCTransform. Don't regress nCount_RNA (already absorbed by NB model). Don't regress S/G2M.Score if studying cell cycle.

🧩 與整合順序

多樣本流程順序：每樣本各自 Normalize → SelectIntegrationFeatures → PrepSCTIntegration → IntegrateData。不要先 merge 再做一次大 SCTransform——會抹除批次資訊。

Multi-sample order: normalize per sample → SelectIntegrationFeatures → PrepSCTIntegration → IntegrateData. Don't merge then SCTransform once — that destroys batch info.

🎯 分清 assay

SCT assay 用於 PCA / Clustering / UMAP；RNA assay (LogNormalize) 用於 FindMarkers / DotPlot / FeaturePlot——這些可視化在 NB residual 上會出現負值，不直觀。

SCT assay → PCA / Clustering / UMAP. RNA assay (LogNormalize) → FindMarkers / DotPlot / FeaturePlot — residuals would show negative values which are hard to interpret in plots.

🔬 物種設定

scale.factor=10000 是 Seurat 預設經驗值，無需更改。percent.mt 偵測：人類 ^MT-、小鼠 ^mt-、斑馬魚 ^mt-，注意大小寫。

scale.factor=10000 is Seurat's default — don't change. percent.mt pattern: human ^MT-, mouse ^mt-, zebrafish ^mt- — mind the case.

🧪 驗證標準化

畫一張校正後 nCount_RNA vs. PC1/PC2。若 PC1 仍強烈與 depth 相關，代表標準化不夠；換 SCTransform 或加入回歸。

Plot post-norm nCount_RNA vs. PC1/PC2. If PC1 still correlates strongly with depth, normalization failed — switch to SCTransform or regress more.

陷阱

七、常見陷阱

⚠️
最常見的 5 個錯誤：
① 對已 SCTransform 的對象再跑 NormalizeData——雙重標準化會把信號壓得失真。
② 在 FindMarkers 時用 SCT assay——SCT v1 對 DE 不友善；應切回 RNA assay 並重新 NormalizeData。
③ 先 merge 多樣本後再 SCTransform——應每樣本各自 SCT，再用 PrepSCTIntegration。
④ vars.to.regress 加入 nCount_RNA——已被模型吸收，重複校正會 over-correct。
⑤ 用 scale.factor=1e6（CPM 慣性）——Seurat 預設 1e4 不要動。
Top 5 mistakes:
① Running NormalizeData on an SCTransformed object — double normalization distorts signal.
② Calling FindMarkers on SCT assay — SCT v1 is not DE-friendly; switch to RNA assay and re-run NormalizeData.
③ Merging samples then SCTransforming — should SCT per sample then PrepSCTIntegration.
④ Adding nCount_RNA to vars.to.regress — already absorbed by the model; double-correction is harmful.
⑤ Using scale.factor=1e6 (CPM habit) — Seurat default 1e4 should not be changed.

🚨 Cluster 按 depth 分群

原因：LogNormalize 沒清除深度影響。
解決：改用 SCTransform，或檢查是否漏做 ScaleData，或 PCA 把第一個 PC 排除（DimPlot 從 PC2 開始）。

Cause: LogNormalize didn't fully remove depth effect.
Fix: Switch to SCTransform, ensure ScaleData was run, or skip PC1 in DimPlot.

🚨 SCT theta 失敗

原因：細胞數太少（< 200）或極稀疏。
解決：用 LogNormalize 或 min.cells 提高基因過濾門檻。

Cause: Too few cells (< 200) or extreme sparsity.
Fix: Use LogNormalize or raise min.cells filter.

🚨 v1 vs v2 差異

原因：v2 使用 glmGamPoi 與不同 dispersion 估計策略。
建議：新項目用 v2；若延續舊分析需鎖定版本 (vst.flavor='v1')。

Cause: v2 uses glmGamPoi with different dispersion strategy.
Suggestion: Use v2 for new projects; if continuing legacy, pin vst.flavor='v1'.

🚨 FindMarkers 空結果

原因：使用了 SCT assay，但 SCT residuals 不適合 DE。
解決：DefaultAssay(obj) <- "RNA"，再執行 NormalizeData() 後做 FindMarkers。

Cause: SCT assay used, but residuals aren't DE-friendly.
Fix: DefaultAssay(obj) <- "RNA", run NormalizeData(), then FindMarkers.

🚨 整合後 SCT 出錯

原因：未跑 PrepSCTIntegration() 或 PrepSCTFindMarkers()。
解決：整合前後分別加上對應 Prep* 函數。

Cause: Missing PrepSCTIntegration() or PrepSCTFindMarkers().
Fix: Add the corresponding Prep* function before/after integration.

🚨 Scran size factor 負值

原因：pool 太小或細胞過度稀疏。
解決：增大 quickCluster 的 min.size、提高 min.mean，或改用 LogNormalize。

Cause: Pool too small or cells too sparse.
Fix: Increase quickCluster's min.size, raise min.mean, or fall back to LogNormalize.

A. 它移除了所有 0 值A. It removes all zeros
B. 它讓所有基因都標準化到 [0,1]B. It scales every gene to [0,1]
