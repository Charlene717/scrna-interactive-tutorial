STEP 8 / 9

細胞註釋 (Cell Annotation)

根據 marker genes、生物學背景與文獻資訊，為 cluster 指定細胞類型名稱。

Assign cell type identities to clusters using marker genes, biological knowledge, and literature references.

概覽

一、從 Cluster 到細胞身份

聚類產生的 cluster 編號（0、1、2…）只是數學上的分群結果，本身並不告訴你每個群代表什麼細胞。細胞註釋 (Annotation) 的任務，是將這些抽象的數字標籤轉換成具有生物學意義的細胞類型名稱（如 CD4+ T cell、CD14 monocyte、basal epithelium 等）。

為什麼這是整個流程中最關鍵也最容易出錯的步驟？因為註釋結果直接決定了你後續所有生物學解讀的正確性。如果你錯把 NK 細胞當作 CD8+ T 細胞，那麼後續的細胞通訊分析、軌跡推斷、差異表現都會建立在錯誤的基礎上。

目前有三大主流策略：

  • ① Marker-based 手動註釋：透過 FindAllMarkers 找出每個 cluster 的特異性基因，再對照文獻、CellMarker、PanglaoDB 等資料庫人工判斷
  • ② Reference-based 自動註釋：使用 SingleR、scType、CellTypist、Azimuth 等工具，將你的數據與已建立的參考數據集做相似度比對
  • ③ Label transfer（標籤遷移）：基於 Seurat anchors 或 scANVI，將參考數據集的細胞標籤「投影」到你的查詢數據

The cluster numbers from clustering (0, 1, 2…) are merely mathematical groupings — they don't tell you what cell type each cluster represents. Cell annotation converts these abstract labels into biologically meaningful identities (CD4+ T cell, CD14 monocyte, basal epithelium, etc.).

Why is this the most critical and error-prone step? Because annotation results directly determine the correctness of all downstream biological interpretation. If you mislabel NK cells as CD8+ T cells, every subsequent cell-communication, trajectory, and DE analysis will be built on a flawed foundation.

Three mainstream strategies dominate the field:

  • ① Marker-based manual annotation: Use FindAllMarkers to discover cluster-specific genes, then manually match against literature, CellMarker, PanglaoDB, etc.
  • ② Reference-based automated annotation: Tools like SingleR, scType, CellTypist, Azimuth compare your data with curated reference datasets
  • ③ Label transfer: Seurat anchors or scANVI project reference labels onto your query data
💡
黃金原則：絕對不要只依賴一種方法。最佳實踐是至少使用 2 種獨立方法（例如：SingleR 自動 + 手動 marker 驗證），看結果是否一致。若不一致，就要回頭檢查 marker 表達、QC、聚類解析度等可能原因。 Golden rule: Never rely on a single method. Best practice uses at least 2 independent approaches (e.g., SingleR auto + manual marker validation) and checks for consistency. If they disagree, revisit marker expression, QC, and clustering resolution.
三大策略

二、註釋方法的三大策略

每種策略各有適用情境與盲點，理解其底層邏輯才能在實務中靈活選擇。

Each strategy has its strengths and blind spots — understanding the underlying logic helps you pick the right one.

🔬

① Marker-based 手動

邏輯：對每個 cluster 跑差異表達分析（DEA），找出 top markers，再對照文獻判斷。

優點：對罕見細胞、新亞群最敏感；不依賴外部參考；可發現新生物學。

缺點：耗時、主觀；需要豐富領域知識；組織越陌生越困難。

典型工作流： FindAllMarkers → 取 top 5-10 markers → 比對 CellMarker / PanglaoDB → 由專家判斷。

Logic: Run DEA for each cluster, get top markers, then judge identity from literature.

Pros: Most sensitive to rare populations and novel subtypes; no external reference needed; can find new biology.

Cons: Time-consuming, subjective; needs deep domain knowledge; difficult for unfamiliar tissues.

Typical flow: FindAllMarkers → top 5-10 markers → check CellMarker / PanglaoDB → expert call.

🤖

② Reference-based 自動

邏輯：將每個細胞（或 cluster）的表達譜與大型參考數據集中的「典型細胞」做相似度比對，挑選最相似的標籤。

優點：快速、可重複、客觀；適合常見組織（PBMC、肺、腦皮層）。

缺點：受參考品質限制；若資料集中沒有的細胞（如腫瘤亞群）會被強制歸入錯誤類別。

代表工具：SingleR、scType、CellTypist、scClassify。

Logic: Compare each cell (or cluster) profile to "canonical" cells in a large reference set and pick the most similar label.

Pros: Fast, reproducible, objective; great for common tissues (PBMC, lung, cortex).

Cons: Bounded by reference quality; cells absent from the reference (e.g., tumor subtypes) get forced into the wrong category.

Examples: SingleR, scType, CellTypist, scClassify.

🔗

③ Label transfer 遷移

邏輯：用一個帶標籤的參考數據集（query-reference 模型），透過 anchor / latent space 找出兩者的對應細胞，將標籤「遷移」到 query。

優點：能同時整合與標註；保留參考的細胞層級結構；適用於 atlas-style 比對。

缺點：需要高品質、結構良好的參考；對未見過的細胞類型仍有「強制歸類」風險。

代表工具：Seurat FindTransferAnchors、Azimuth、scANVI、scArches。

Logic: Use a labeled reference, find anchor cells (or shared latent space) between query and reference, then transfer the labels.

Pros: Integrates and annotates simultaneously; preserves reference hierarchy; suits atlas-style mapping.

Cons: Needs a high-quality, well-structured reference; unseen cell types are still force-classified.

Examples: Seurat FindTransferAnchors, Azimuth, scANVI, scArches.

Marker 資料庫

四、Marker 資料庫

無論用哪種策略，最終都需要查閱權威的細胞類型 marker 資料庫。以下是公認的五大寶庫：

Whichever strategy you use, you'll need authoritative cell-type marker databases. The community relies on these five:

📕 CellMarker 2.0

整理自 ~10,000 篇文獻，涵蓋 ~2,000 種人與小鼠細胞類型、~26,000 個 markers。提供搜尋介面，可按組織或細胞類型查詢。
適用：需要文獻可追溯的 marker；想知道「ALB + AFP 在肝臟代表什麼細胞」這類問題。

Curated from ~10,000 papers, covers ~2,000 human and mouse cell types and ~26,000 markers. Searchable by tissue or cell type.
Use for: Literature-traceable markers; questions like "what cell does ALB + AFP indicate in liver?"

📗 PanglaoDB

提供完整的「細胞類型 → marker gene set」對應表，並已內建在 scType / sc-type 等工具中。對每個 marker 給予 sensitivity 與 specificity 分數。
適用：scType 等基於 marker score 的自動工具；快速取得 cell-type gene signature。

Provides complete "cell-type → marker gene set" mappings, embedded in scType and similar tools. Each marker has sensitivity/specificity scores.
Use for: Score-based auto tools like scType; rapid signature lookups.

📘 Human Cell Atlas (HCA)

跨組織的人類細胞 atlas，提供下載完整 AnnData 物件。可作為 SingleR、scANVI 或 Azimuth 的參考數據集。
適用：需要 atlas-level reference 做 label transfer 或整合。

Cross-tissue human cell atlas, full AnnData objects downloadable. Serves as a reference dataset for SingleR, scANVI, or Azimuth.
Use for: Atlas-level reference for label transfer or integration.

📙 Tabula Sapiens / Muris

由 CZ Biohub 主導，跨組織、跨技術（10x + Smart-seq2）的 human / mouse 多器官 atlas。標註細緻、品質高。
適用：整器官層級的 reference；跨物種比較研究。

CZ Biohub-led, cross-tissue, cross-technology (10x + Smart-seq2) human/mouse atlas. Fine annotation, high quality.
Use for: Organ-level references; cross-species comparison.

📓 MSigDB Cell-Type Signatures (C8)

MSigDB 的 C8 子集合，收錄超過 800 個 cell-type signature gene sets。可直接用於 GSEA、UCell 或 AUCell 計算 module score。
適用：用 signature score 補強單一 marker 的判斷。

MSigDB's C8 collection, 800+ cell-type signature gene sets. Plug into GSEA, UCell, or AUCell for module-level scoring.
Use for: Reinforce single-marker calls with signature scores.

📒 Allen Brain Atlas / Tabula Microcebus

專科 atlas，適合腦科學或非模式生物研究者。
適用：小鼠/獼猴腦、靈長類比較研究等專門領域。

Specialty atlases for neuroscience or non-model organisms.
Use for: Mouse/macaque brain, primate comparative studies, niche fields.

五、註釋置信度模擬器

真實世界裡，marker 並非「全有或全無」——同一個 marker 可能在多種細胞中都有不同程度的表達，這就是 specificity（特異性）。當 marker specificity 低或資料雜訊高時，自動工具給出的置信度分數就會大幅下降，導致「ambiguous」的呼叫。

決策引導

八、手動 vs 自動 決策

🌳 註釋策略決策流程

Q1:
你的組織有良好的公開參考數據集（如 PBMC、肺、腎、腦皮層）嗎？→ 是 → 使用 Azimuth / SingleR + HPCA 做初步自動註釋，再用 FindAllMarkers 手動驗證。
Q2:
沒有合適的參考？→ 純手動：跑 FindAllMarkers，把每個 cluster 的 top markers 對照 CellMarker / PanglaoDB / 文獻判斷。
Q3:
某些 cluster 的 markers 很混亂、找不到典型對應？→ 可能是 ① doublets、② over-clustering（解析度太高）、③ 真正的新亞群、或 ④ 過渡狀態。回頭調整 QC、resolution，或考慮跑 trajectory 分析。
Q4:
有疾病樣本 vs 健康樣本？→ 先在健康樣本完成註釋，再用 label transfer 把標籤轉到疾病樣本，避免疾病細胞被強制歸入錯誤類別。
Q5:
跨物種（人類 vs 小鼠）？→ 用 SingleR 並指定對應物種的 reference（如 ImmGen for mouse）；marker 名稱需轉換 ortholog（如 CD3D → Cd3d）。
Q1:
Good public reference for your tissue (PBMC, lung, kidney, cortex)?→ Yes → Use Azimuth / SingleR + HPCA for first pass, then FindAllMarkers for manual verification.
Q2:
No suitable reference?→ Pure manual: run FindAllMarkers, match top markers per cluster against CellMarker / PanglaoDB / literature.
Q3:
Some clusters have messy/atypical markers?→ Possible causes: ① doublets, ② over-clustering, ③ genuine new subpopulation, or ④ transitional state. Revisit QC, resolution, or run trajectory analysis.
Q4:
Disease vs healthy samples?→ Annotate on healthy first, then label transfer to disease — avoids forcing disease cells into wrong classes.
Q5:
Cross-species (human vs mouse)?→ Use SingleR with species-matched reference (ImmGen for mouse); convert ortholog names (CD3D → Cd3d).

九、實作範例

# ===== 手動 marker-based annotation (Seurat) =====
library(Seurat); library(dplyr)

# Step 1: 找每個 cluster 的 markers
markers <- FindAllMarkers(pbmc,
  only.pos       = TRUE,        # 只看上調基因
  min.pct        = 0.25,        # 至少 25% 細胞表達
  logfc.threshold = 0.25)

# Step 2: 取每個 cluster 的 top 10
top10 <- markers %>%
  group_by(cluster) %>%
  slice_max(order_by = avg_log2FC, n = 10)

# Step 3: 視覺化驗證
FeaturePlot(pbmc, features = c("CD3D", "MS4A1", "CD14", "GNLY"))
DotPlot(pbmc, features = c("CD3D", "IL7R", "CD8A",
                            "MS4A1", "CD14", "GNLY")) + RotatedAxis()

# Step 4: 確認後手動指定細胞類型
new.ids <- c("CD4 T", "CD14 Mono", "B cell",
              "CD8 T", "NK", "DC", "FCGR3A Mono", "Platelet")
names(new.ids) <- levels(pbmc)
pbmc <- RenameIdents(pbmc, new.ids)
pbmc$cell_type <- Idents(pbmc)
# ===== SingleR 自動註釋 =====
library(SingleR); library(celldex)

# 1. 載入參考數據集（HumanPrimaryCellAtlas）
ref <- celldex::HumanPrimaryCellAtlasData()

# 2. 跑 SingleR（cluster-level 模式較穩定）
pred <- SingleR(
  test     = GetAssayData(pbmc, slot = "data"),
  ref      = ref,
  labels   = ref$label.main,
  clusters = Idents(pbmc))    # cluster-level

# 3. 將 SingleR 標籤回填到 Seurat 物件
pbmc$singler_label <- pred$labels[match(Idents(pbmc),
                                          rownames(pred))]

# 4. 檢查 pruning（ambiguous 的 cluster 會被標 NA）
plotScoreHeatmap(pred)
plotDeltaDistribution(pred)

# ===== Azimuth (PBMC reference) =====
library(Azimuth)
pbmc <- RunAzimuth(pbmc, reference = "pbmcref")
# 結果存於 pbmc$predicted.celltype.l1 / l2 / l3
DimPlot(pbmc, group.by = "predicted.celltype.l2", label = TRUE)
# ===== Manual marker-based (Scanpy) =====
import scanpy as sc

# Step 1: 找 marker genes
sc.tl.rank_genes_groups(adata, groupby="leiden",
                         method="wilcoxon")
sc.pl.rank_genes_groups(adata, n_genes=10)

# Step 2: 視覺化
sc.pl.umap(adata, color=["CD3D", "MS4A1", "CD14", "GNLY"])
sc.pl.dotplot(adata,
  var_names=["CD3D", "IL7R", "CD8A",
             "MS4A1", "CD14", "GNLY"],
  groupby="leiden")

# Step 3: 取得 DataFrame 形式的 markers
markers_df = sc.get.rank_genes_groups_df(adata, group=None)

# Step 4: 手動指定
celltype_map = {
  "0": "CD4 T", "1": "CD14 Mono",
  "2": "B cell", "3": "CD8 T",
  "4": "NK", "5": "DC",
}
adata.obs["cell_type"] = adata.obs["leiden"].map(celltype_map)
# ===== CellTypist 自動註釋 =====
import celltypist
from celltypist import models

# 1. 下載模型（首次需聯網）
models.download_models(model="Immune_All_Low.pkl")
model = models.Model.load(model="Immune_All_Low.pkl")

# 2. 預測（建議使用 raw 或標準化的 log1p 數據）
predictions = celltypist.annotate(
  adata,
  model="Immune_All_Low.pkl",
  majority_voting=True)   # 同 cluster 多數決

# 3. 將結果合併回 AnnData
adata = predictions.to_adata()
sc.pl.umap(adata, color=["majority_voting",
                        "predicted_labels"])

# 4. 檢查置信度
adata.obs["conf_score"] = predictions.predicted_labels["conf_score"]
sc.pl.violin(adata, ["conf_score"], groupby="leiden")
最佳實務

十、最佳實務

✅ 交叉驗證

至少使用 2 種獨立方法（如 SingleR + 手動 markers），比較結果。一致 → 高信心；不一致 → 標記為「需專家判斷」並深入檢查。

Use at least 2 independent methods (e.g., SingleR + manual markers) and cross-check. Agreement → high confidence; disagreement → flag for expert review.

✅ 人工檢查 top 3

即使有自動標籤，務必親眼確認每個 cluster 的 top 3 markers 是否符合該細胞類型的經典標誌。FeaturePlot + VlnPlot 同時看分佈與強度。

Even with auto labels, eyeball the top 3 markers of each cluster against canonical signatures for that cell type. Combine FeaturePlot + VlnPlot for distribution and intensity.

✅ 區分 Unknown vs Novel

找不到對應的 cluster 不一定是「新發現」。先排除：技術假象、過度聚類、doublets。確認後再思考是否為真正的新生物學。

An unrecognised cluster is not automatically a "discovery." First rule out: technical artefacts, over-clustering, doublets. Then consider novel biology.

✅ 記錄參考版本

SingleR 的 reference（如 HPCA v2024.01）、Azimuth 的 reference 版本、CellTypist 的 model 名稱與下載日期，全部記錄到 metadata。可重現性的關鍵。

Log the SingleR reference version (e.g., HPCA v2024.01), Azimuth reference release, CellTypist model name + download date — all in metadata. Key to reproducibility.

✅ 保留階層標籤

不要只存最細的標籤。同時保留 L1（T cell）、L2（CD4 T）、L3（CD4 memory）。不同分析（trajectory vs DE）可能用不同層級。

Don't keep only the finest label. Preserve L1 (T cell), L2 (CD4 T), L3 (CD4 memory). Different downstream analyses (trajectory vs DE) need different granularity.

✅ 加註信心等級

在 metadata 加 annotation_confidence（high / medium / low）。後續分析時可選擇排除 low confidence cluster，避免污染結論。

Add an annotation_confidence column (high/medium/low) in metadata. Downstream you can exclude low-confidence clusters to avoid contaminating conclusions.

常見陷阱

十一、常見陷阱

🚨 陷阱 1：盲信 SingleR 對罕見細胞的呼叫

SingleR 等基於參考的工具只能從參考集中已有的類別中選。如果你的數據裡有 reference 從未看過的細胞（例如新的腫瘤亞群、罕見 progenitor），SingleR 還是會「硬塞」一個最像的標籤，並可能給出看似合理的 score。對策：檢視 plotScoreHeatmap，若該 cluster 對多個標籤的 score 都很低或差距很小（low delta），應視為 ambiguous 並改用手動判斷。

Reference-based tools like SingleR can only choose from labels present in the reference. For cells the reference has never seen (novel tumor subsets, rare progenitors), SingleR still assigns the closest match — possibly with a plausible-looking score. Mitigation: inspect plotScoreHeatmap; if the cluster scores low across labels or has low delta between top picks, treat as ambiguous and use manual judgment.

🚨 陷阱 2：單一 marker 即下結論

看到 CD3D 表達就標 T 細胞、看到 CD14 就標 monocyte——這是初學者最常犯的錯。單一 marker 永遠不夠，因為：① 表達是連續的，背景值可能被誤判為陽性；② 一個基因可能在多種細胞中都有表達（CD4 也存在於部分 monocyte）。正確做法：至少 3 個 markers 同時陽性 + 至少 1 個應為陰性的 marker 驗證（如 NK 應 CD3D⁻）。

Calling T cells from CD3D alone, or monocytes from CD14 alone, is the most common beginner mistake. Single markers are never enough: ① expression is continuous and background may be miscalled positive; ② a gene can be expressed across multiple cell types (CD4 appears in some monocytes). Best practice: require ≥3 markers positive + ≥1 expected-negative marker (e.g., NK should be CD3D-negative).

🚨 陷阱 3：過度套用 Azimuth 參考

Azimuth 的 PBMC reference 是在健康人類 PBMC 上訓練的。如果你做的是類風濕關節炎滑液、癌症淋巴結或骨髓瘤骨髓等病理樣本，Azimuth 仍會給出 PBMC 細胞類型，但可能完全錯過病理特有的細胞狀態。對策：把 Azimuth 視為「baseline 框架」，再用手動 markers 補強疾病特有訊號；或選擇對應疾病的專門 reference（CellTypist 有 COVID、IBD 等模型）。

Azimuth's PBMC reference was trained on healthy human PBMC. For RA synovial fluid, cancer lymph node, or myeloma bone marrow samples, Azimuth will still hand out PBMC labels — but may completely miss disease-specific states. Mitigation: treat Azimuth as a "baseline scaffold," then augment with manual markers for disease-specific signals; or pick disease-specific references (CellTypist offers COVID, IBD, etc.).

🚨 陷阱 4：物種不匹配

用人類 HPCA 參考分析小鼠數據是常見錯誤。小鼠基因名稱（Cd3d 而非 CD3D）、表達譜差異、免疫亞群定義都不同。對策：小鼠請用 celldex::ImmGenData() 或 MouseRNAseqData()；跨物種比較時，需先轉換 ortholog（biomaRt 或 homologene）。

Using the human HPCA reference for mouse data is a frequent mistake. Mouse gene symbols (Cd3d not CD3D), expression patterns, and immune subset definitions all differ. Mitigation: use celldex::ImmGenData() or MouseRNAseqData() for mouse; for cross-species, convert orthologs (biomaRt or homologene).

🚨 陷阱 5：忽略過渡 / 中間狀態

並非所有 cluster 都對應「離散」細胞類型。在發育、活化、分化途中的細胞，可能同時表達兩個類型的 markers。強行單一標籤會抹去重要的生物學動態。對策：使用 trajectory / pseudotime 分析；或保留 "CD4_T_activating" 這類過渡標籤；或將該 cluster 標為 "transitional"。

Not every cluster maps to a discrete cell type. Cells mid-development, activation, or differentiation may co-express markers from two types. Forcing a single label erases important dynamics. Mitigation: use trajectory / pseudotime analysis; keep transitional labels like "CD4_T_activating"; or mark the cluster as "transitional".
