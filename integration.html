ADVANCED

數據整合 (Data Integration)

整合多樣本或多批次資料，降低 batch effect 並保留真實生物學差異。

Integrate multi-sample or multi-batch data — remove batch effects while preserving true biological differences.

概覽

一、為什麼需要 Integration？

合併不同實驗批次的 scRNA-seq 數據時，最棘手的問題是：聚類結果可能反映「樣本來源」而不是「細胞類型」。每次實驗的系統性差異（試劑批號、操作手法、定序平台、實驗日期、不同個體基因型）會在 UMAP 上產生「按樣本分群」的偽結果，掩蓋真正的生物學訊號。

Integration 的目標：

  • 同一細胞類型跨不同樣本應互相重疊（batch mixed）
  • 不同細胞類型仍應分開聚類（biology preserved）
  • 避免「過度校正」把真正的生物學差異也抹平

判斷需不需要 Integration 的快速診斷：

  • UMAP 用 sample / batch 著色 → 若顏色分明、不混合 → 需要 Integration
  • 用 cell type 著色 → 若同類型細胞被切成多個小團 → 需要 Integration
  • 用 sample 著色 → 若顏色完全混在一起 → 不需要 Integration（已無 batch effect）

The hardest problem when combining scRNA-seq batches is: clustering may reflect "sample origin" rather than "cell type." Systematic differences (reagent lots, operator technique, sequencing platform, experiment date, donor genotype) produce "clusters-by-sample" on UMAP and mask the true biology.

Goals of integration:

  • The same cell type across samples should overlap (batch mixed)
  • Different cell types should still cluster apart (biology preserved)
  • Avoid "over-correction" that erases real biological differences

Quick diagnostic for whether integration is needed:

  • Colour UMAP by sample / batch → distinct, non-mixing colours → integration needed
  • Colour by cell type → same type broken into multiple sub-clusters → integration needed
  • Colour by sample → colours fully mixed → integration NOT needed (no batch effect)
💡
關鍵原則：Integration 不是「越強越好」。理想狀態是 batch 混合 + cell type 分開。過度校正會把腫瘤特有細胞與正常細胞「合併」、把不同分化階段「壓平」。每次 integration 後都要同時看 batch 與 cell-type 著色的 UMAP。 Key principle: Integration is NOT "more is better." The ideal state is batch mixed + cell types separated. Over-correction merges tumor-only populations with normals and flattens differentiation states. Always inspect UMAPs coloured by both batch and cell type after integration.
批次效應來源

二、批次效應的常見來源

理解 batch 來源，才能在實驗設計階段避免無法挽回的混淆，並在分析時選擇對應的策略。

Understanding the sources of batch effects helps you avoid irreparable confounding at design time and choose the right strategy at analysis.

🧪

① 試劑與化學版本

10x Chromium v2 vs v3：v3 偵測敏感度與 UMI 數約高 1.5–2 倍；Smart-seq2 vs 10x：full-length vs 3'-bias 差異巨大。同一物種、同一細胞，不同化學版本的基因偵測譜可能完全不同。

10x Chromium v2 vs v3: v3 has 1.5–2× higher sensitivity and UMI counts; Smart-seq2 vs 10x: full-length vs 3'-biased capture differs dramatically. Same species, same cell — totally different detection profiles.

📅

② 捕獲日期與操作者

即使試劑相同，不同天、不同操作者也會產生顯著差異。樣本解離溫度、時間、細胞活性都影響基因表達譜。冷藏隔夜 vs 立即跑會大幅改變壓力反應基因。

Even with identical reagents, different days / operators produce significant differences. Dissociation temperature, time, and cell viability all affect expression. Overnight storage vs immediate run drastically changes stress-response genes.

🧬

③ 個體間變異

不同病人 / 小鼠的基因型、性別、年齡、免疫狀態都不同。這些差異在 PBMC 等高度動態的組織中尤其明顯。即使技術完全相同，個體間 batch 仍存在。

Donors differ in genotype, sex, age, and immune state. Effects are especially strong in dynamic tissues like PBMC. Even with identical tech, donor-level batch effects remain.

🏥

④ 跨實驗室 / 中心

合作研究或公開數據集 meta-analysis 時，不同實驗室的 SOP、儀器型號、定序深度都不同。是最強的 batch 來源之一。

In collaborative studies or public-data meta-analysis, different labs use different SOPs, instruments, and sequencing depths. One of the strongest batch sources.

🔄

⑤ 多時間點 / 多條件

同一研究內不同時間點 / 不同處理組，若沒有交叉設計，時間點與條件會混淆。複雜批次結構（multi-batch × multi-condition）需要更強大的方法（如 scVI）。

Across timepoints / treatment groups in the same study, without crossed design, timepoint confounds with condition. Complex batch structure (multi-batch × multi-condition) needs stronger tools (e.g., scVI).

💻

⑥ 比對流程 / 參考基因組

不同 Cell Ranger 版本、不同基因組 build（GRCh38 vs GRCh37）、不同 GTF 註釋，會造成同一細胞的 UMI matrix 略有差異。Meta-analysis 公開數據時特別要注意。

Different Cell Ranger versions, genome builds (GRCh38 vs GRCh37), or GTF annotations produce slightly different UMI matrices for the same cell. Watch out in meta-analyses of public data.

四大方法家族

三、四大整合方法家族

主流方法可分為四大類，各有適用情境與限制：

Mainstream methods fall into four families, each with its niche:

🔗

① CCA-based

代表：Seurat v3 IntegrateData (CCA + anchors)。
邏輯：用 Canonical Correlation Analysis 找跨批次共享的相關結構，定義「anchor cells」（在不同批次中互為最近鄰），用 anchors 校正其他細胞。
優：能保留細胞類型結構；對小批次效應穩定。
缺：記憶體與計算成本高；批次間細胞組成差異大時容易出錯。

Examples: Seurat v3 IntegrateData (CCA + anchors).
Logic: Canonical Correlation Analysis finds shared correlation structure across batches; defines "anchor cells" (mutual nearest neighbours) and corrects others through anchors.
Pros: preserves cell-type structure; stable for small batch effects.
Cons: memory & compute hungry; fails when cell composition differs greatly.

🤝

② MNN-based

代表：mnnCorrect、fastMNN（Bioconductor batchelor）。
邏輯：找出跨批次的「互為最近鄰」(Mutual Nearest Neighbours) 細胞對；以這些 MNN 對為「橋」，校正整個矩陣。
優：不需事前定義 anchors；對細胞組成差異有一定容忍度。
缺：對極端不對稱的批次可能殘留 batch；速度中等。

Examples: mnnCorrect, fastMNN (Bioconductor batchelor).
Logic: Find mutual nearest neighbours across batches; use them as "bridges" to correct the matrix.
Pros: no anchors needed up front; tolerates moderate composition differences.
Cons: may leave residual batch with very asymmetric batches; moderate speed.

⚡

③ Embedding-based

代表：Harmony（最廣泛使用）、scanorama。
邏輯：在 PCA 嵌入空間中，迭代地把每個細胞「拉向」同類細胞的中心、同時遠離批次中心。最終輸出一個校正過的低維空間（不是 expression matrix）。
優：極快、記憶體小；對大資料集（百萬細胞）也能跑；最廣泛使用的方法。
缺：輸出只在嵌入空間有效——不能用於 DE。

Examples: Harmony (most widely used), scanorama.
Logic: In PCA embedding space, iteratively pull each cell toward the centre of same-type cells and away from batch centroids. Output is a corrected low-dim space, not an expression matrix.
Pros: very fast, low memory; scales to millions of cells; most widely adopted.
Cons: correction lives only in embedding space — not usable for DE.

🧠

④ Deep learning

代表：scVI、scANVI、scArches、totalVI（含 CITE-seq）。
邏輯：用變分自編碼器 (VAE) 學一個 latent space，明確將 batch 作為 conditional variable 分離出來。scANVI 額外加入 cell-type label 監督。
優：處理複雜批次結構；可儲存模型供新數據投影（scArches）；同時支援多模態。
缺：需 GPU；訓練時間長；hyperparameter 調整較困難；對 hyperparameter 敏感。

Examples: scVI, scANVI, scArches, totalVI (with CITE-seq).
Logic: A variational autoencoder learns a latent space, with batch as a conditional variable explicitly separated. scANVI adds cell-type label supervision.
Pros: handles complex batch structure; reusable models for new-data projection (scArches); multi-modal support.
Cons: needs GPU; long training; sensitive to hyperparameters.

互動模擬

五、整合強度滑桿

調整下方的「整合強度」滑桿，觀察兩個 batch × 三種細胞類型在 2D 空間的位置變化。重點觀察過度整合的後果：當強度過高時，原本只在 batch B 中存在的「rare cell type」會被強行併入其他細胞，生物學訊號被抹平。

Move the "integration strength" slider and watch two batches × three cell types in 2D. Note the consequence of over-integration: at high strength, a rare cell type present only in batch B gets forced into other cells — biology is erased.

— — —

0 = 未整合；0.4-0.6 = 最佳區；> 0.85 = 過度整合

⚠️
觀察重點：① 強度 0–0.3 區間 batch 仍可見，bio 訊號完整；② 0.4-0.6 是「甜蜜點」——batch 混合且 cell type 仍分開；③ > 0.8 時 batch B 獨有的稀有細胞會被「吃掉」，無法挽回。實務上 Harmony / scVI 的預設參數通常落在甜蜜點，但每次都要看著 UMAP 確認，不要盲信預設值。 Key insights: ① At 0–0.3, batch is still visible, biology intact; ② 0.4-0.6 is the "sweet spot" — batches mixed, cell types distinct; ③ Above 0.8, the rare batch-B-only population is absorbed — unrecoverable. In practice Harmony / scVI defaults usually sit in the sweet spot, but always inspect UMAPs — never trust defaults blindly.

程式碼

七、實作範例

# ===== Harmony (R) — 最常用 =====
library(Seurat); library(harmony)

# Step 1: 整合前的標準流程（每個 sample 各自 HVG → 合併）
pbmc <- merge(pbmc1, list(pbmc2, pbmc3))
pbmc <- NormalizeData(pbmc) %>%
        FindVariableFeatures() %>%
        ScaleData() %>%
        RunPCA(npcs = 30)

# Step 2: 跑 Harmony（在 PCA 空間做校正）
pbmc <- RunHarmony(pbmc,
  group.by.vars = "batch",
  reduction     = "pca",
  dims.use      = 1:30,
  plot_convergence = TRUE)

# Step 3: 下游分析以 harmony reduction 為主
pbmc <- FindNeighbors(pbmc, reduction = "harmony", dims = 1:30)
pbmc <- FindClusters(pbmc, resolution = 0.5)
pbmc <- RunUMAP(pbmc, reduction = "harmony", dims = 1:30)

# Step 4: 視覺驗證（同時看 batch 與 cell type）
DimPlot(pbmc, group.by = "batch")        # 應充分混合
DimPlot(pbmc, group.by = "cell_type")    # 應仍分群
# ===== Seurat v3/v4 CCA-based Integration =====
library(Seurat)

obj.list <- list(b1 = pbmc1, b2 = pbmc2, b3 = pbmc3)
obj.list <- lapply(obj.list, function(x) {
  x <- NormalizeData(x)
  x <- FindVariableFeatures(x, nfeatures = 2000)
})

# Step 1: 選共同 HVG
features <- SelectIntegrationFeatures(obj.list, nfeatures = 2000)

# Step 2: 找 anchors（CCA + MNN）
anchors <- FindIntegrationAnchors(obj.list,
  anchor.features = features,
  reduction = "cca")

# Step 3: 整合
integ <- IntegrateData(anchorset = anchors)
DefaultAssay(integ) <- "integrated"

# Step 4: 標準下游
integ <- ScaleData(integ) %>% RunPCA() %>%
         RunUMAP(dims = 1:30) %>%
         FindNeighbors(dims = 1:30) %>%
         FindClusters()

# ⚠️ DE 要回到 RNA assay！
DefaultAssay(integ) <- "RNA"
# ===== Seurat v5 IntegrateLayers (RPCA) =====
library(Seurat)

# Step 1: 將 RNA assay 依 batch split 成多個 layers
pbmc[["RNA"]] <- split(pbmc[["RNA"]], f = pbmc$batch)

# Step 2: 標準預處理
pbmc <- NormalizeData(pbmc)
pbmc <- FindVariableFeatures(pbmc)
pbmc <- ScaleData(pbmc)
pbmc <- RunPCA(pbmc)

# Step 3: RPCA 整合（細胞組成差異大時優於 CCA）
pbmc <- IntegrateLayers(
  object         = pbmc,
  method         = RPCAIntegration,
  orig.reduction = "pca",
  new.reduction  = "integrated.rpca",
  verbose        = TRUE)

# Step 4: 整合後重新合併 layers + 下游
pbmc[["RNA"]] <- JoinLayers(pbmc[["RNA"]])
pbmc <- FindNeighbors(pbmc, reduction = "integrated.rpca", dims = 1:30)
pbmc <- RunUMAP(pbmc, reduction = "integrated.rpca", dims = 1:30)
# ===== scVI (Python) — 深度學習 VAE =====
import scvi
import scanpy as sc

# Step 1: 準備 AnnData，告訴 scVI 哪個欄位是 batch
scvi.model.SCVI.setup_anndata(adata,
  batch_key = "batch",
  layer     = "counts",        # 必須是 raw counts!
  categorical_covariate_keys = ["donor"])

# Step 2: 建立並訓練模型
model = scvi.model.SCVI(adata,
  n_latent  = 30,
  n_layers  = 2,
  gene_likelihood = "nb")
model.train(max_epochs = 200,
  early_stopping = True,
  early_stopping_patience = 20)

# Step 3: 取得 latent representation
adata.obsm["X_scVI"] = model.get_latent_representation()

# Step 4: 下游用 X_scVI
sc.pp.neighbors(adata, use_rep = "X_scVI")
sc.tl.leiden(adata, resolution = 0.5)
sc.tl.umap(adata)
sc.pl.umap(adata, color = ["batch", "leiden"])

# 加碼：scANVI（有 cell-type label 監督）
lvae = scvi.model.SCANVI.from_scvi_model(model,
  adata             = adata,
  labels_key        = "cell_type",
  unlabeled_category = "Unknown")
lvae.train(max_epochs = 50)
adata.obsm["X_scANVI"] = lvae.get_latent_representation()
# ===== Harmonypy (Python 版的 Harmony) =====
import scanpy.external as sce
import scanpy as sc

# Step 1: 預處理
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
sc.pp.highly_variable_genes(adata, n_top_genes=2000, batch_key="batch")
sc.pp.scale(adata, max_value=10)
sc.tl.pca(adata, n_comps=30)

# Step 2: Harmony
sce.pp.harmony_integrate(adata, key="batch",
  adjusted_basis="X_pca_harmony",
  max_iter_harmony=20)

# Step 3: 下游用 X_pca_harmony
sc.pp.neighbors(adata, use_rep="X_pca_harmony")
sc.tl.leiden(adata, resolution=0.5)
sc.tl.umap(adata)
sc.pl.umap(adata, color=["batch", "leiden"])
品質評估

八、品質評估指標

整合後不能只看 UMAP 的「感覺」。應使用量化指標檢查 batch mixing 與 biology preservation 兩個維度。

Don't judge by UMAP "feel" alone. Use quantitative metrics for both batch mixing and biology preservation.

🔀 kBET (k-nearest-neighbour Batch Effect Test)

對每個細胞檢驗 k 個最近鄰中各 batch 的比例是否符合期望（卡方檢定）。低 kBET rejection rate = 批次混合良好。
適合：分散異質的批次組成。

For each cell, test whether the batch composition of its k nearest neighbours matches expected (chi-square). Low kBET rejection = good batch mixing.
Best for: heterogeneous batch compositions.

📐 LISI (Local Inverse Simpson's Index)

iLISI（batch LISI）越高 → 局部 batch 多樣性越好；cLISI（cell-type LISI）越低 → 局部 cell-type 一致性越好。理想：iLISI ↑ 同時 cLISI ↓。
Harmony 論文用此為主要指標。

iLISI (batch LISI) high → good local batch diversity; cLISI (cell-type LISI) low → good local cell-type consistency. Ideal: iLISI ↑ AND cLISI ↓.
Used as the main metric in the Harmony paper.

📊 ASW (Average Silhouette Width)

對 batch 計算 ASW（越接近 0 越好——表示 batch 不形成獨立 cluster）；對 cell-type 計算 ASW（越高越好——表示 cell-type 仍分開）。

ASW for batch (close to 0 = better — batch doesn't form its own cluster); ASW for cell-type (higher = better — types still distinct).

🏆 scIB Benchmark

Luecken et al. 2022 (Nat Methods) 提供整合的綜合 benchmark，計算 14 種指標的加權分數，分為 batch correction 與 biology conservation 兩大類。
金標準的整合評估框架，每次發表前建議跑。

Luecken et al. 2022 (Nat Methods) provides a comprehensive benchmark combining 14 metrics into batch-correction + biology-conservation scores.
The gold-standard evaluation framework; run before publication.

最佳實務

九、最佳實務

✅ 分批選 HVG，取聯集

正確流程：每個 sample 各自計算 HVG → 取聯集（union 或 intersection）。直接在合併後的數據選 HVG 會被最大 batch 主導，丟失小 batch 特有的訊號。Seurat SelectIntegrationFeatures 與 scanpy highly_variable_genes(batch_key=...) 都支援此邏輯。

Correct flow: compute HVGs per sample → take union (or intersection). Selecting HVGs on the merged matrix is dominated by the largest batch and loses small-batch-specific signals. Seurat SelectIntegrationFeatures and scanpy highly_variable_genes(batch_key=...) both support this.

✅ QC 在 Integration 之前

低品質細胞、MT% 過高的死細胞、未清除的 doublets，會被 integration 算法當成「batch 結構」處理，造成嚴重的偽校正。務必先 QC 乾淨，再做 integration。

Low-quality cells, high-MT% dying cells, and uncleaned doublets are treated as "batch structure" by integration algorithms — leading to bogus corrections. Always QC clean first, then integrate.

✅ 從 Harmony 開始

Harmony 速度快、預設參數穩定、廣泛使用。第一輪一律 Harmony，效果不佳再考慮 RPCA / scVI。不要一上來就用最重的工具。

Harmony is fast, has stable defaults, and is widely accepted. Always Harmony first; only escalate to RPCA / scVI if Harmony fails. Don't bring out the heavy artillery up front.

✅ DE 必須用 RNA assay

整合後的 integrated assay 或 X_scVI latent 都不可用於 DE——它們已經被「校正」，每個基因的數值不再有量化意義。DE 一律回到 DefaultAssay = "RNA" 並用 pseudo-bulk。

The integrated assay or X_scVI latent cannot be used for DE — values have been "corrected" and lose quantitative meaning. Always revert to DefaultAssay = "RNA" + pseudo-bulk for DE.

✅ 設計階段就避免 batch-condition 混淆

每個 batch 應同時含有 disease + control（balanced design）。若 batch 與 condition 完全混淆（所有 disease 在 batch A、所有 control 在 batch B），沒有任何整合方法能挽救——分析時要明確聲明 limitation。

Each batch should contain BOTH disease and control (balanced design). If batch fully confounds with condition (all disease in batch A, all control in batch B), no integration method can save you — flag as a limitation.

✅ 註釋後不再 re-integrate

完成 annotation 後，不要再回頭重做 integration——這會打亂已建立的 cluster 邊界，需要重新註釋。如果要加入新樣本，使用 scArches / FindTransferAnchors 等 reference mapping 把新數據「投影」到既有空間。

After annotation, do not re-integrate — it scrambles existing cluster boundaries and forces re-annotation. To add new samples, use scArches / FindTransferAnchors reference mapping to project them onto the existing space.

常見陷阱

十、常見陷阱

🚨 陷阱 1：過度整合抹平生物學

若一個 cell type 真的只存在於某個條件（如腫瘤細胞只在疾病樣本中），強行整合會把它與其他細胞「合併」。對策：整合後用 cell-type 著色檢查；若疾病特有 cluster 不見了，調降整合強度（Harmony 的 theta 由 2 降至 1）或改用 RPCA / scANVI（後者用 label 監督避免 over-correction）。

If a cell type genuinely exists in only one condition (tumour cells in disease only), forced integration merges it with other cells. Mitigation: after integration, colour by cell type; if a disease-only cluster vanishes, reduce integration strength (Harmony's theta from 2 → 1) or switch to RPCA / scANVI (latter uses label supervision to prevent over-correction).

🚨 陷阱 2：未過濾 MT / RPS 即整合

粒線體基因、核糖體基因在不同 batch 間可能有大幅差異（樣本處理 stress 不同），未過濾就整合會讓算法把 MT/RPS 差異當成最大 batch 訊號去校正，反而抑制真正的細胞類型分群。對策：QC 時用 PercentageFeatureSet 過濾；HVG 選擇時排除 ^MT- 與 ^RPS/^RPL。

MT and ribosomal genes can vary hugely across batches (different dissociation stress). Integrating without filtering them makes the algorithm chase MT/RPS variance as the dominant batch signal, suppressing real cell-type structure. Mitigation: filter with PercentageFeatureSet in QC; exclude ^MT-, ^RPS/^RPL from HVG selection.

🚨 陷阱 3：輸入有 NaN / Inf

合併多個 sample 時若有空 column、log(0) 未處理、scaling 後出現 NaN，scVI / Harmony 會在訓練中爆掉或產生奇怪結果。對策：跑 integration 前用 any(is.na(...)) / np.isnan(adata.X).sum() 確認；移除全零基因；scVI 一定要傳 raw counts 而非 log1p。

When merging samples, empty columns, unhandled log(0), or NaN after scaling can crash scVI / Harmony or produce odd results. Mitigation: verify with any(is.na(...)) / np.isnan(adata.X).sum(); drop all-zero genes; pass raw counts to scVI, not log1p.

🚨 陷阱 4：批次與條件完全混淆

若所有 disease 樣本在 batch A 跑、所有 control 在 batch B，整合必然連 disease 效應一起移除——沒有方法能分離兩者。對策：實驗設計階段每個 batch 內混合條件；若數據已收集且混淆，不應做 cross-condition 整合分析，或在論文中明確聲明此 limitation。

If all disease samples are in batch A and controls in batch B, integration inevitably removes disease effect along with batch — no method can separate them. Mitigation: balance condition across batches at design time; if data is already confounded, do not perform cross-condition integration analysis, or declare it as a limitation.

🚨 陷阱 5：用 IntegratedData 做 DE

Seurat integrated assay 或 scVI X_scVI 經過校正，不是原始 expression 值。在它上面跑 FindMarkers 或 pseudo-bulk 會得到嚴重失真的結果（已知最常見的單細胞分析錯誤之一）。對策：DE 前明確 DefaultAssay(obj) <- "RNA"；Python 用 adata.layers["counts"]。

Seurat's integrated assay or scVI's X_scVI contain corrected, not raw, values. Running FindMarkers or pseudo-bulk on them gives severely distorted results (one of the most common known mistakes in single-cell analysis). Mitigation: explicitly set DefaultAssay(obj) <- "RNA" before DE; in Python use adata.layers["counts"].

🚨 陷阱 6：scVI / scANVI 訓練不收斂

scVI 訓練時 loss 不下降、ELBO 振盪：常見原因是 ① 傳入 log1p 而非 raw counts；② batch_key 設定錯誤；③ 樣本量太小（< 5000 細胞）；④ early stopping patience 太短。對策：檢查 setup_anndata 用的 layer；用 plot_convergence=True 監控；確認 raw counts；增加 epochs 與 patience。

scVI fails to converge — loss won't drop, ELBO oscillates. Common causes: ① passing log1p instead of raw counts; ② wrong batch_key; ③ too few cells (<5000); ④ early-stopping patience too tight. Mitigation: check the layer in setup_anndata; monitor with plot_convergence=True; confirm raw counts; increase epochs and patience.
