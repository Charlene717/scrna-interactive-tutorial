STEP 7 / 9

UMAP 視覺化

將高維資料投影到 2D 給人眼觀察——這是視覺化工具，不是分析空間。

Project high-dim data to 2D for the human eye — this is a visualization tool, not an analysis space.

概覽

一、UMAP 是視覺化，不是分析

UMAP (Uniform Manifold Approximation and Projection, McInnes 2018) 是非線性降維方法，把 PCA 空間中的細胞關係投影到 2D 平面，讓人類視覺能「一眼看出」細胞的群結構。

但 UMAP 在 scRNA-seq 流程裡的角色經常被誤解。請先記住三個關鍵分工：

1. 分群在 PCA 空間做（kNN→Louvain/Leiden 用的是前 N 個 PC）
2. 細胞類型註解用 marker 基因驗證（不是看 UMAP 上的位置）
3. UMAP 只是「呈現結果的圖」——用於人類視覺溝通

UMAP (Uniform Manifold Approximation and Projection, McInnes 2018) is a non-linear dimensionality reduction method that projects cell relationships from PCA space to a 2D plane — letting human vision "see at a glance" the cluster structure.

UMAP's role in scRNA-seq pipelines is frequently misunderstood. Remember three divisions of labor:

1. Clustering happens in PCA space (kNN → Louvain/Leiden runs on the top N PCs)
2. Cell-type annotation is validated by marker genes (not by UMAP position)
3. UMAP is just "the figure that presents results" — for human visual communication
🚨
嚴重誤讀警告：
① UMAP 圖上群體間的距離不代表生物學相似度
② 群體大小（佔據面積）不代表細胞數量
③ 群體形狀與軸向會因隨機種子改變
④ UMAP 上看似的「軌跡」不是真正的分化軌跡（請用 Slingshot / PAGA / Monocle3）
這些都是非線性投影的副產品，絕不能直接做生物學推論。 Severe misinterpretation warning:
① Inter-cluster distances on UMAP do NOT represent biological similarity
② Cluster size (area on plot) does NOT represent cell counts
③ Cluster shapes / axes change with random seed
④ Apparent "trajectories" on UMAP are NOT real differentiation trajectories (use Slingshot / PAGA / Monocle3)
These are artifacts of non-linear projection — never use them for biological inference.
數學直覺

二、UMAP 數學直覺

UMAP 的核心想法是 「保持資料的局部流形結構」。流程可拆成兩階段：

  1. 建構高維模糊圖：在 PCA 空間中為每個細胞找最近的 n_neighbors 個鄰居；用模糊集合（fuzzy simplicial set）表示「i 與 j 是否屬於同一鄰域」的不確定性。鄰域大小由局部密度自適應決定。
  2. 找低維對應圖：在 2D 中初始化點（通常用 spectral embedding），然後用梯度下降最小化高維模糊圖與低維模糊圖之間的交叉熵。min_dist 控制低維中點與點能擠得多近。

為什麼比 t-SNE 好？ t-SNE 的成本函數（KL 散度）只懲罰「鄰居被分開」，不懲罰「遠處被拉近」——所以全局結構被破壞。UMAP 的交叉熵雙向懲罰——既保留局部鄰居，也避免遠處不當靠近，全局結構保留得較好。

UMAP's core idea is preserving the local manifold structure of the data. The pipeline has two stages:

  1. Build a fuzzy graph in high-dim: in PCA space, find each cell's nearest n_neighbors; use a fuzzy simplicial set to represent the uncertainty of "do i and j share a neighborhood?" Neighborhood size adapts to local density.
  2. Find a low-dim counterpart: initialize points in 2D (typically via spectral embedding), then use gradient descent to minimize the cross-entropy between the high-dim and low-dim fuzzy graphs. min_dist controls how tightly points can pack in low-dim.

Why better than t-SNE? t-SNE's cost (KL divergence) only penalizes "neighbors pulled apart," not "far points pulled close" — destroying global structure. UMAP's cross-entropy penalizes both directions, preserving local neighbors AND preventing inappropriate long-range collapse — global structure survives better.

📍

局部 vs 全局

UMAP 對「同類細胞的鄰近性」保留得最好；群與群之間的相對距離則只有有限的可信度——千萬別把它當成生物分化距離。

UMAP best preserves "within-type proximity"; between-cluster distances are only of limited reliability — never read them as biological divergence distances.

🎲

非確定性

UMAP 的初始化、優化都涉及隨機。不同 random_state 會得到不同的旋轉、鏡像、區域形狀——必須固定 seed 才能可重現。

UMAP's initialization and optimization are stochastic. Different random_state values give different rotations, mirrorings, and local shapes — fix the seed for reproducibility.

關鍵參數

三、四個關鍵參數

🎯 n_neighbors （預設 15–30）

意義：高維鄰居數，決定關注「局部 vs 全局」結構。
小值（5–10）：放大局部結構；群分得很開；細微亞型容易現形，但全局可能被打散。
大值（50+）：更平滑、全局結構連續，但稀有族群可能被吞掉。

Meaning: number of high-dim neighbors; balances "local vs global" structure.
Small (5–10): amplifies local structure; clusters separate widely; fine subtypes emerge but global structure may fragment.
Large (50+): smoother, more continuous global structure, but rare populations may be absorbed.

📏 min_dist （預設 0.3–0.5）

意義：低維空間中點與點允許的最小距離。
小值（0.0–0.1）：群體緊密、結構清晰，適合分群展示。
大值（0.5–1.0）：群體較鬆散、漸變區明顯，適合展示細胞狀態連續性。

Meaning: minimum permitted distance between points in low-dim.
Small (0.0–0.1): tight, well-defined clusters — good for cluster figures.
Large (0.5–1.0): looser clusters with visible gradients — good for showing cell-state continuums.

↔️ spread （預設 1.0）

意義：整體投影的「展開度」。spread 越大，整張圖被拉得越開。實務上很少調整——保持預設即可。

Meaning: overall "spread" of the projection. Larger spread → the whole layout is stretched out. Rarely tuned in practice — keep default.

📐 metric （預設 cosine / euclidean）

scRNA-seq 慣例：Scanpy 在 sc.pp.neighbors 預設用 euclidean（在 scaled PCA 空間中）；Seurat 預設 cosine。對 sparse log-normalized 資料，cosine 通常表現更好——它對 library size 不敏感。

scRNA-seq convention: Scanpy uses euclidean by default in sc.pp.neighbors (on scaled PCA space); Seurat defaults to cosine. For sparse log-normalized data, cosine usually performs better — it's insensitive to library size.

💡
調參順序：先 n_neighbors（決定看局部還是全局），再 min_dist（決定群有多緊湊），其他保持預設。最大忌諱：每跑一次就調一次參數——你會無意識地調到「結果好看」而非「結果正確」。 Tuning order: first tune n_neighbors (local vs global), then min_dist (cluster tightness); leave the rest at defaults. Worst habit: tweaking parameters every run — you will subconsciously drift toward "pretty" rather than "correct."

程式碼

六、實作範例

# 標準 UMAP——使用 30 個 PC
pbmc <- RunUMAP(pbmc, dims = 1:30,
               n.neighbors = 30,
               min.dist = 0.3,
               metric = "cosine",
               seed.use = 42)

# 視覺化分群結果
DimPlot(pbmc, reduction = "umap", label = TRUE)

# 視覺化 marker gene 表達（FeaturePlot 是驗證細胞類型的核心工具）
FeaturePlot(pbmc, features = c("CD3D", "MS4A1", "CD14", "GNLY"))

# 用 sample / batch 上色檢查批次效應
DimPlot(pbmc, reduction = "umap", group.by = "orig.ident")

# 探索性：嘗試不同 min.dist（不可在最終分析中反覆嘗試！）
pbmc <- RunUMAP(pbmc, dims = 1:30, min.dist = 0.1,
               reduction.name = "umap_tight")
pbmc <- RunUMAP(pbmc, dims = 1:30, min.dist = 0.8,
               reduction.name = "umap_loose")
DimPlot(pbmc, reduction = "umap_tight")
DimPlot(pbmc, reduction = "umap_loose")
import scanpy as sc

# 先建鄰居（這一步決定 n_neighbors）
sc.pp.neighbors(adata, n_pcs=30, n_neighbors=15, metric="cosine")

# 跑 UMAP
sc.tl.umap(adata, min_dist=0.3, spread=1.0, random_state=42)

# 視覺化 cluster
sc.pl.umap(adata, color=["leiden"], legend_loc="on data")

# 視覺化 markers
sc.pl.umap(adata, color=["CD3D", "MS4A1", "CD14", "GNLY"])

# 檢查批次效應
sc.pl.umap(adata, color=["sample"])

sc.tl.umap(adata, min_dist=0.3, method="densmap")
⚠️
seed.use / random_state 一定要固定！不固定 seed 等於每次跑 UMAP 都是「新的 UMAP」——形狀會變、論文圖無法重現、reviewer 重跑會得不同結果。論文補充材料應記錄 seed、n_neighbors、min_dist、dims、metric 全部參數。 Always fix seed.use / random_state! An unfixed seed means every run produces a "different UMAP" — shape varies, paper figures can't be reproduced, reviewers re-running get different results. Supplementary materials should list seed, n_neighbors, min_dist, dims, and metric.
最佳實務

七、最佳實務

✅ 固定 random_state

論文圖、與 reviewer 重現實驗都需要可重複。Seurat seed.use=42、Scanpy random_state=42——選一個並記錄。

Paper figures and reviewer reproducibility require it. Seurat seed.use=42, Scanpy random_state=42 — pick one and document it.

✅ 用 FeaturePlot 驗證

分群完後，把 canonical marker 疊在 UMAP 上：CD3D（T）、MS4A1（B）、CD14（Mono）。Marker 落在哪一群就是哪一群。不要用 UMAP 上的位置去推測類型。

After clustering, overlay canonical markers on UMAP: CD3D (T), MS4A1 (B), CD14 (Mono). Whichever cluster the marker lands on is that type. Never infer type from UMAP position alone.

✅ 不要跨 UMAP 比較

兩個資料集分別跑 UMAP 後，座標不可直接比較（連同一資料的兩次運行都不可比）。如要比較，用 marker 表達、cluster 比例，不要靠 UMAP 上的位置。

UMAPs from different datasets are not directly comparable (even two runs on the same data aren't). For comparisons, use marker expression and cluster proportions — not UMAP coordinates.

✅ 比較圖參數一致

要在論文 figure 中並列多個條件（treated vs control），務必用相同 n_neighbors、min_dist、seed 重跑，避免讀者誤把參數差異當成生物學差異。

For figures juxtaposing multiple conditions (treated vs control), rerun with identical n_neighbors, min_dist, and seed — preventing readers from misreading parameter differences as biology.

✅ dims 與聚類一致

UMAP 用的 dims 應該與 FindNeighbors/sc.pp.neighbors 用的 n_pcs 一致——這樣 UMAP 才正確反映分群結果所在的空間。

UMAP's dims should match the n_pcs used in FindNeighbors/sc.pp.neighbors — so the UMAP correctly reflects the space in which clusters were defined.

✅ 多樣本先整合

不整合就直接跑 UMAP，多半第一眼就看到「樣本分開」。先 Harmony / scVI 整合得到 batch-corrected embedding，再以該 embedding 跑 UMAP。

Without integration, you'll typically see samples separating at first glance. Run Harmony / scVI integration to get a batch-corrected embedding, then run UMAP on that.

常見陷阱

八、常見陷阱

📏 📏 距離 ≠ 相似度

誤讀：「cluster A 和 cluster B 在 UMAP 上很近，所以它們生物學上很像。」
真相：UMAP 距離只能模糊代表「同類細胞密度高」。跨群距離高度受參數和 seed 影響。
正解：用 marker 共表達、Pearson 相關、Wilcoxon DE 評估相似度。

Misread: "Cluster A and B are close on UMAP, so they're biologically similar."
Reality: UMAP distance only loosely encodes "within-type density." Inter-cluster distance is heavily parameter- and seed-dependent.
Fix: assess similarity via marker co-expression, Pearson correlation, Wilcoxon DE.

🛤️ 🛤️ 軌跡誤讀

誤讀：看到 UMAP 上一條長帶就說「這是分化軌跡」。
真相：UMAP 沒有方向、長度也無意義。所謂「軌跡」可能只是兩個鄰群被拉成一條。
正解：用 Slingshot、PAGA、Monocle3 做正式 pseudotime 推論。

Misread: a long band on UMAP → "this is a differentiation trajectory."
Reality: UMAP has no direction; length is meaningless. The "trajectory" may just be two adjacent clusters stretched into a band.
Fix: use Slingshot, PAGA, or Monocle3 for formal pseudotime.

🧪 🧪 批次效應主導

症狀：UMAP 顯示樣本分開、每個 island 對應一個 donor。
解法：必須先 Harmony / RPCA / scVI integration，不能靠調 UMAP 參數來「修」批次。

Symptom: UMAP shows separated samples, each island = one donor.
Fix: integrate first (Harmony / RPCA / scVI). You cannot "fix" batch by tweaking UMAP params.

🎨 🎨 過度調參

陷阱：反覆改 min_dist、n_neighbors，最後選「最漂亮」的圖。
問題：這是結果偏差——你選的不是最忠實呈現，而是最符合你預期的。
解法：用預設值或一次性決定，並在論文中誠實揭露。

Trap: repeatedly tweaking min_dist / n_neighbors and picking the "prettiest."
Problem: this is outcome bias — you're not selecting the most faithful but the most confirmation-friendly.
Fix: use defaults or commit once; disclose honestly in the paper.

📌 📌 邊界過度解讀

誤讀：「兩群中間有些重疊的點，所以這些細胞處於過渡狀態。」
真相：UMAP 邊界是優化副產品，不一定有生物學意義。
正解：對這些細胞做 DE 分析、檢查 marker 是否真的混合。

Misread: "Overlapping points at the boundary = transitional state."
Reality: UMAP boundaries are optimization artifacts, not necessarily biological.
Fix: run DE on those border cells, check whether markers truly mix.

🪞 🪞 不同跑次不一樣

狀況：同一資料跑兩次 UMAP，圖像旋轉、鏡像、形狀都變了。
原因：UMAP 的優化是隨機梯度下降。
解法：固定 random_state；論文圖固定後不要再重跑。

Situation: running UMAP twice on the same data gives different rotations, mirrors, and shapes.
Cause: UMAP's optimization is stochastic gradient descent.
Fix: fix random_state; once your paper figure is set, don't re-run.

⚠️
UMAP 的「三不」：不量距離、不看面積、不解讀軌跡。它只回答一個問題——「我的資料分成幾群、看起來大致長什麼樣」。任何精細的生物學結論都必須回到 PCA 空間、marker 基因、統計檢定來支持。 UMAP's three "don'ts": don't measure distances, don't read areas, don't interpret trajectories. It answers exactly one question — "roughly how many groups are there and what do they look like?" Any fine biological conclusion must be supported by PCA space, marker genes, and statistical tests.
