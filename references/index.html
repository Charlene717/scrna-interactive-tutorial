勘誤

⚠️ 對原教學內容的疑義注記

下方為閱讀原教學 HTML 與本 reference 比對後，發現的可能不完整、易誤解或可加強之處。不修改原教學檔案，僅在此說明以利參照。

QC：DoubletFinder API 名稱

教學中示範 doubletFinder_v3()。此為 DoubletFinder < 2.0.4 的舊 API；自 2023 年發行的 DoubletFinder 2.0.4 起，函式已正名為 doubletFinder() 與 paramSweep()（不再帶 _v3 後綴）。新版本仍向後相容舊名稱，但建議讀者使用新 API。

來源：GitHub: chris-mcginnis-ucsf/DoubletFinder 的 README 與 release notes。

QC：Doublet 偵測工具名單

原教學列出 DoubletFinder / Scrublet / scDblFinder 三個工具。根據 Xi & Li 2021 Cell Systems 的獨立 benchmark，scDblFinder 在多數測試情境表現最佳，DoubletFinder 與 Scrublet 仍為廣泛使用工具但精度稍遜，可作為三者並列時的補充說明。

來源：Xi & Li, Cell Syst 2021。

Annotation：FCGR3A Mono 的常用名

教學表格寫「FCGR3A Mono」，FCGR3A 即 CD16；在多數教科書、Seurat 官方 PBMC tutorial 中，此細胞群通常稱為 CD16+ Monocyte 或 非典型單核球（non-classical monocyte）。兩者指向同一細胞群，名稱可互換。

來源：Hao et al. 2021 Cell（Azimuth PBMC reference）；Seurat PBMC3K vignette。

Annotation：DC marker 的細分

教學表格使用 FCER1A、CST3 標示 DC。實際上：

· CST3 為廣義骨髓系（myeloid）/單核細胞與 DC 共表達基因，特異性較低；
· FCER1A 主要標示 cDC2（conventional DC type 2）；
· cDC1 常用 marker 為 CLEC9A、XCR1；
· pDC（漿細胞樣 DC）常用 marker 為 LILRA4、IL3RA。
若要精細註釋 DC 子群，建議搭配 Azimuth PBMC reference 或 CellTypist 的 immune model 使用。

來源：Domínguez Conde et al., Science 2022。

DE：跨條件比較的當前共識

教學提到「Pseudobulk + DESeq2」是跨條件最佳實踐。Squair et al. 2021 與 Murphy et al. 2022 進一步指出：把 cluster × sample pseudobulk 後使用 edgeR-LRT 或 DESeq2 LRT，比直接對單細胞做 Wilcoxon / MAST 在控制 FDR 上明顯較佳；同時應特別留意樣本數過少時 pseudobulk 也不可靠。

來源：Squair et al. 2021；Murphy et al. 2022。

UMAP：警告應更明確

教學已指出「UMAP 兩 cluster 距離近不代表分子相似」，可進一步補充 Chari & Pachter 2023 的觀點：2D embedding 距離有結構性失真，應避免將 UMAP 座標用於下游量化分析或最終生物學結論。分群、軌跡推論應在 PCA / 高維 embedding 上進行，UMAP 僅作可視化。

來源：Chari & Pachter 2023。

Normalization：「正式分析推薦 SCTransform」應併呈 benchmark

教學中將 SCTransform 列為「目前主流推薦」。Ahlmann-Eltze & Huber 2023 Nat Methods 在 transformations 比較中發現：對於下游 KNN/聚類效果，shifted log + size factor（即 LogNormalize 的變體）與 SCTransform Pearson residuals 在許多任務表現相當，差距小於主觀印象。建議讀者依實驗目的測試比較。

來源：Ahlmann-Eltze & Huber 2023。

Clustering：Seurat 預設 k=20

教學中提及「Seurat 預設 k=20」。技術上正確：Seurat FindNeighbors() 的 k.param 預設值為 20。但建議補充：此值會隨資料規模與細胞異質性調整，例如極大資料集（>100K cells）或細胞群極多時，可能需要更大的 k 值來維持圖的連通性。

來源：Seurat FindNeighbors 文件。
