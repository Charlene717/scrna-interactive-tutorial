差異表達分析

比較不同 cluster 或 condition 的基因表達差異，找出 markers 與疾病訊號。

Compare gene expression differences between clusters or conditions to find markers and disease signals.

概覽

一、scRNA-seq DE 的兩大情境

差異表達分析 (Differential Expression, DE) 是 scRNA-seq 從「描述細胞」走到「解釋機制」的橋樑。它有兩種完全不同的應用情境：

  • ① Cluster-vs-cluster DE：找出某個 cluster 相對其他細胞的特異性基因（marker genes），用於細胞註釋。
  • ② Condition-vs-condition DE：比較同一細胞類型在不同實驗條件下（疾病 vs 健康、藥物 vs 對照、不同時間點）的表達差異，用於機制研究。

為什麼 scRNA-seq DE 與 bulk RNA-seq 完全不同？單細胞數據有三大特殊挑戰：

  • ① 零膨脹 (Zero Inflation)：技術性 dropout 加上低 UMI，使每個細胞有 80–95% 的基因偵測為 0。傳統 t-test、limma 假設常態分佈完全不成立。
  • ② 龐大樣本量 (Inflated n)：一個樣本可能有上萬細胞。把每個細胞當獨立觀察會嚴重虛胖統計檢力，幾乎所有基因都會「顯著」。
  • ③ 細胞間依存 (Pseudo-replication)：來自同一個體的細胞並非獨立——它們共享基因型、處理批次、技術變異。

因此，scRNA-seq DE 不能直接套用 bulk 方法，需要特別設計的工具或策略。最重要的兩種策略是 per-cell DE（Wilcoxon、MAST、LR）與 pseudo-bulk DE（DESeq2、edgeR、limma-voom）。

Differential expression (DE) is the bridge from "describing cells" to "explaining mechanism" in scRNA-seq. It serves two distinct scenarios:

  • ① Cluster-vs-cluster DE: find cluster-specific genes (markers) for cell annotation.
  • ② Condition-vs-condition DE: compare the same cell type across conditions (disease vs healthy, drug vs control, timepoints) for mechanistic studies.

Why is scRNA-seq DE completely different from bulk RNA-seq? Three unique challenges:

  • ① Zero inflation: technical dropouts and low UMI give 80–95% zeros per cell. Classical assumptions of t-test / limma break down.
  • ② Inflated n: a sample can contain 10,000+ cells. Treating each as independent dramatically inflates statistical power — virtually every gene becomes "significant."
  • ③ Cellular pseudo-replication: cells from the same donor are NOT independent — they share genotype, batch, technical variation.

scRNA-seq DE cannot apply bulk methods directly. Two key strategies dominate: per-cell DE (Wilcoxon, MAST, LR) and pseudo-bulk DE (DESeq2, edgeR, limma-voom).

💡
核心原則：不同情境用不同工具。
① 找 markers → Wilcoxon 即可。
② 跨條件比較 + 多個生物重複 → pseudo-bulk + DESeq2（目前最佳實踐，Squair et al. 2021 證實）。
③ 只有單樣本 → MAST 或 Wilcoxon，但統計效力有限，結論需保守。 Core principle: different scenarios, different tools.
① Find markers → Wilcoxon works.
② Cross-condition with biological replicates → pseudo-bulk + DESeq2 (current best practice, Squair et al. 2021).
③ Single-sample only → MAST or Wilcoxon, but power is limited — conclusions must be conservative.
三大檢定方法

二、三大檢定方法家族

📊

① Wilcoxon Rank-Sum

假設：無分佈假設（非參數）。比較兩組秩次。

優點：對 outlier 與分佈形狀不敏感、計算極快、Seurat FindMarkers 預設方法。

缺點：無法調整 covariate（年齡、性別、批次）；無法處理生物重複的層級結構。

適用：cluster markers、初步探索。

Assumption: distribution-free (non-parametric). Compares ranks.

Pros: robust to outliers, very fast, Seurat FindMarkers default.

Cons: cannot adjust covariates (age, sex, batch); does not handle hierarchical replication.

Use: cluster markers, exploratory work.

🪝

② MAST (Hurdle GLM)

假設：表達量服從「是否表達 (Bernoulli) + 表達量 (Gaussian)」的雙階段模型（hurdle）。專為單細胞零膨脹設計。

優點：能加入 covariate（如 cellular detection rate、批次）；明確分離零值與非零值的訊號。

缺點：運算較慢；仍是 per-cell 模型，依然有 pseudo-replication 風險。

適用：單樣本內跨條件比較；需要控制 covariate 時。

Assumption: hurdle model — a Bernoulli for "is expressed" + Gaussian for expression level. Designed for single-cell zero inflation.

Pros: can add covariates (CDR, batch); explicitly separates zero vs non-zero signals.

Cons: slower; still a per-cell model — pseudo-replication risk remains.

Use: within-sample cross-condition; when covariate adjustment is needed.

🧮

③ Pseudo-bulk + DESeq2 / edgeR

假設：先把每個 sample × cell type 的 UMI 加總成一個「假性 bulk」樣本，再用標準 bulk RNA-seq 方法（負二項分佈）分析。

優點：正確處理生物重複；嚴格控制 false positive；可加入完整設計矩陣（age、sex、batch）。

缺點：需要至少 3 個生物重複；單樣本不可用；會喪失細胞層級的細微訊號。

適用：跨條件比較且有多個生物重複——這是目前 scRNA-seq DE 的最佳實踐。

Assumption: aggregate UMIs per sample × cell type into "pseudo-bulk" samples, then apply standard bulk RNA-seq methods (negative binomial).

Pros: correctly handles biological replication; strict FDR control; supports full design matrix (age, sex, batch).

Cons: needs ≥3 biological replicates; not usable for single-sample; loses cell-level fine signals.

Use: cross-condition with biological replicates — the current best practice.

Pseudo-bulk

三、Pseudo-bulk 深入

2018 年 Soneson & Robinson 在 Nature Methods 系統比較了 36 種 DE 方法，發現傳統 per-cell 方法在跨條件 DE 時 false discovery 嚴重失控。隨後 Squair et al. 2021（Nature Communications, "Confronting false discoveries in single-cell differential expression"）進一步指出：

  • 把每個細胞當獨立樣本的 Wilcoxon / MAST / t-test，會將真正的樣本層級變異當成統計效力，產生大量 false positives。
  • 當每個條件有多個生物重複時，把同一條件中所有細胞的 UMI 加總（pseudo-bulk），再用 DESeq2 / edgeR / limma-voom 分析，控制偽陽性的能力遠優於 per-cell 方法。
  • 這個結論在多個 benchmark（包括 muscat 框架）上都得到驗證。

📐 Pseudo-bulk 工作流

  1. 分組：對每個 cell type 分別處理。
  2. 聚合：對該 cell type 內，每個 sample 的細胞 UMI 直接相加，得到 genes × samples 的矩陣。
  3. 過濾：移除細胞數 < 10 的 sample × cell type 組合（樣本量太小）。
  4. 標準化：DESeq2 的 size factor 或 edgeR 的 TMM。
  5. 檢定：標準 bulk DE（負二項 GLM + Wald / LRT）。

In 2018, Soneson & Robinson (Nature Methods) systematically compared 36 DE methods and found that traditional per-cell methods have catastrophically inflated FDR for cross-condition DE. Squair et al. 2021 (Nature Communications, "Confronting false discoveries in single-cell differential expression") sharpened the conclusion:

  • Per-cell Wilcoxon / MAST / t-test confuse sample-level variation with statistical power, generating many false positives.
  • With biological replicates per condition, aggregating UMIs across cells per sample (pseudo-bulk) then running DESeq2 / edgeR / limma-voom controls false discoveries far better than per-cell methods.
  • This holds across multiple benchmarks (including the muscat framework).

📐 Pseudo-bulk workflow

  1. Stratify: process each cell type separately.
  2. Aggregate: sum UMIs across cells within each sample for that cell type → genes × samples matrix.
  3. Filter: drop sample × cell-type combinations with <10 cells.
  4. Normalise: DESeq2 size factors or edgeR TMM.
  5. Test: standard bulk DE (negative binomial GLM with Wald / LRT).
⚠️
關鍵警告：Pseudo-bulk 不是萬靈丹——它必須有真正的生物重複。如果你的「重複」其實是同一個體的技術重複（同一血液樣本分兩管定序），這仍然是 pseudo-replication。生物重複必須代表獨立的生物學單位（不同個體、不同小鼠）。一般建議每組至少 3 個生物重複，最好 5 個以上。 Critical warning: Pseudo-bulk is no panacea — it requires true biological replicates. If your "replicates" are technical splits of the same individual (one blood sample split into two), this is still pseudo-replication. Biological replicates must represent independent units (different individuals, different mice). At least 3 per group is the floor; 5+ is preferable.

程式碼

七、實作範例

# ===== Wilcoxon (Seurat 預設) — 找 cluster markers =====
library(Seurat)

# 找 cluster 0 vs 其餘的 markers
cl0_markers <- FindMarkers(pbmc,
  ident.1         = 0,
  test.use        = "wilcox",
  only.pos        = TRUE,
  min.pct         = 0.25,
  logfc.threshold = 0.25)

# 一次找所有 cluster 的 markers
all_markers <- FindAllMarkers(pbmc, only.pos = TRUE)

# 過濾 + 排序
top <- all_markers %>%
  filter(p_val_adj < 0.05) %>%
  group_by(cluster) %>%
  slice_max(avg_log2FC, n = 10)
# ===== MAST — 單樣本內跨條件比較 =====
library(Seurat); library(MAST)

# 在 CD4 T cell 中比較 disease vs control
cd4 <- subset(pbmc, cell_type == "CD4 T")
Idents(cd4) <- "condition"   # 切換 ident 為條件

de <- FindMarkers(cd4,
  ident.1         = "disease",
  ident.2         = "control",
  test.use        = "MAST",
  latent.vars     = c("nCount_RNA"),  # CDR adjustment
  logfc.threshold = 0.1)

# 視覺化（火山圖）
library(EnhancedVolcano)
EnhancedVolcano(de,
  lab = rownames(de),
  x = "avg_log2FC", y = "p_val_adj",
  pCutoff = 0.05, FCcutoff = 0.5)
# ===== Pseudo-bulk + DESeq2 — 金標準跨條件 DE =====
library(Seurat); library(DESeq2)

# Step 1: 對每個 sample × cell_type 聚合 UMI
pseudo <- AggregateExpression(pbmc,
  group.by    = c("cell_type", "sample_id"),
  assays      = "RNA",
  slot        = "counts",
  return.seurat = FALSE)

# 取 CD4 T 的 pseudobulk 矩陣
cd4_bulk <- pseudo$RNA[, grep("CD4 T_", colnames(pseudo$RNA))]

# Step 2: 建立 metadata + DESeq2 物件
meta <- data.frame(
  row.names = colnames(cd4_bulk),
  condition = ifelse(grepl("D", colnames(cd4_bulk)), "disease", "control"),
  sex       = c(...)  # 若有 covariate
)
dds <- DESeqDataSetFromMatrix(
  countData = cd4_bulk,
  colData   = meta,
  design    = ~ sex + condition)

# Step 3: 過濾低表達（pseudobulk 後仍需要）
keep <- rowSums(counts(dds) >= 10) >= 3
dds <- dds[keep, ]

# Step 4: 跑 DESeq2 標準流程
dds <- DESeq(dds)
res <- results(dds, contrast = c("condition", "disease", "control"))
summary(res)
res_sig <- subset(res, padj < 0.05 & abs(log2FoldChange) > 0.5)
# ===== Scanpy: Wilcoxon for marker genes =====
import scanpy as sc

# 找所有 cluster 的 markers
sc.tl.rank_genes_groups(
  adata,
  groupby="leiden",
  method="wilcoxon",
  use_raw=False,
  pts=True)

# 視覺化
sc.pl.rank_genes_groups(adata, n_genes=15)
sc.pl.rank_genes_groups_dotplot(adata, n_genes=5)
sc.pl.rank_genes_groups_heatmap(adata, n_genes=5)

# 取得 DataFrame
df = sc.get.rank_genes_groups_df(adata, group="0")
df = df[(df.pvals_adj < 0.05) & (df.logfoldchanges > 0.5)]

# 跨條件比較（不建議直接用，僅作示範）
sc.tl.rank_genes_groups(
  adata[adata.obs.cell_type == "CD4 T"],
  groupby="condition",
  reference="control",
  method="wilcoxon")
# ===== pertpy: Pseudo-bulk in Python =====
import pertpy as pt
import scanpy as sc
import decoupler as dc

# Step 1: 聚合成 pseudo-bulk AnnData
pdata = dc.get_pseudobulk(
  adata,
  sample_col="sample_id",
  groups_col="cell_type",
  layer="counts",
  mode="sum",
  min_cells=10,
  min_counts=1000)

# Step 2: 篩選 CD4 T 並建立 DESeq2 物件
cd4_pdata = pdata[pdata.obs.cell_type == "CD4 T"].copy()

# Step 3: 用 pertpy 的 PyDESeq2 wrapper
dds = pt.tl.PyDESeq2(
  adata=cd4_pdata,
  design_factors=["sex", "condition"],
  refit_cooks=True)
dds.fit()
results = dds.test_contrasts(
  contrast=["condition", "disease", "control"])

# Step 4: 顯著基因
sig = results[(results.padj < 0.05) &
              (results.log2FoldChange.abs() > 0.5)]
決策引導

八、決策樹

🌳 該選哪種方法？

情境 1:
找 cluster markers（為了註釋）→ Wilcoxon（Seurat FindAllMarkers 預設）。快、夠用、廣為接受。
情境 2:
跨條件比較，有 ≥ 3 個生物重複/組？→ Pseudo-bulk + DESeq2（金標準，FDR 控制最好）。
情境 3:
跨條件比較，但只有 1-2 個樣本/組？→ MAST（可加 CDR、批次 covariate），但須明確標註統計效力有限、結論需保守。
情境 4:
需控制連續 covariate（如年齡、病程）？→ Pseudo-bulk + DESeq2（design = ~ age + condition），或 MAST 的 latent.vars。
情境 5:
細胞數極大（>100k）且只想初步看大方向？→ Wilcoxon 配合嚴格 logFC + adj-p 門檻，再用 pseudo-bulk 對重點 genes 驗證。
Case 1:
Cluster markers (for annotation)→ Wilcoxon (Seurat FindAllMarkers default). Fast, sufficient, widely accepted.
Case 2:
Cross-condition with ≥ 3 biological replicates per group?→ Pseudo-bulk + DESeq2 (gold standard, best FDR control).
Case 3:
Cross-condition with only 1-2 samples per group?→ MAST (with CDR / batch covariates), but flag the limited power and keep conclusions conservative.
Case 4:
Need to adjust continuous covariates (age, disease stage)?→ Pseudo-bulk + DESeq2 (design = ~ age + condition) or MAST latent.vars.
Case 5:
Massive cell count (>100k) for a quick scan?→ Wilcoxon with strict logFC + adj-p thresholds, then validate top hits with pseudo-bulk.
最佳實務

九、最佳實務

✅ 同時報 logFC + adj-p + pct

只看 p-value 容易被大樣本量誤導。logFC ≥ 0.5 才有生物學意義，pct.1 / pct.2 確認該基因確實在多數細胞中表達——三者一起才是嚴謹的 marker。

p-value alone is misled by large n. logFC ≥ 0.5 for biological meaning, plus pct.1 / pct.2 showing the gene is actually expressed in most cells — all three matter.

✅ 用 min.pct 預過濾

Seurat FindMarkers 預設 min.pct = 0.1、logfc.threshold = 0.25。這預過濾掉「兩組幾乎都不表達」的基因，大幅減少 multiple testing 負擔，提升真陽性訊號比。

Seurat's FindMarkers defaults: min.pct = 0.1, logfc.threshold = 0.25. These pre-filter genes barely expressed in either group, cutting multiple-testing burden and boosting signal-to-noise.

✅ 重複數 > 細胞數

跨條件 DE 的真正統計效力來自生物重複數，不是細胞數。3 個樣本 × 1000 細胞 ≫ 1 個樣本 × 100,000 細胞。實驗設計時優先增加重複。

For cross-condition DE, real statistical power comes from biological replicates, not cell count. 3 samples × 1000 cells ≫ 1 sample × 100,000 cells. Plan experiments accordingly.

✅ 人工驗證 Top 結果

Top 10-20 markers 一定要用 VlnPlot 或 FeaturePlot 視覺確認分佈合理（不是只有少數細胞拉抬均值），並查文獻是否符合生物學預期。

Always inspect top 10-20 markers with VlnPlot or FeaturePlot for sensible distribution (not just a few outlier cells inflating the mean) and check against literature.

✅ 分群來自批次嗎？

跨條件 DE 前，務必檢查同一條件的多個樣本是否在 UMAP 上混合良好。若一個樣本獨佔某 cluster，pseudobulk 結果其實是「樣本效應」而非「條件效應」。

Before cross-condition DE, verify samples from the same condition mix well on UMAP. If one sample dominates a cluster, your pseudo-bulk result is actually a "sample effect" not a "condition effect".

✅ 移除環境 RNA 污染

環境 RNA（如組織 lysis 釋放的 ALB）會在所有 cluster 都呈現「表達」。先用 SoupX 或 CellBender 清理，避免將高豐度污染基因誤判為跨類型 marker。

Ambient RNA (e.g., ALB leaked from lysed hepatocytes) appears expressed in every cluster. Clean with SoupX or CellBender first to avoid mis-calling abundant contaminants as universal markers.

常見陷阱

十、常見陷阱

🚨 陷阱 1：把細胞當獨立重複

這是 scRNA-seq DE 最嚴重也最常見的錯誤——直接拿 1 個疾病樣本 vs 1 個健康樣本的所有細胞跑 FindMarkers。所謂的「2,000 vs 2,000 細胞」其實只代表各 1 個體，但 Wilcoxon 把它們當 4,000 個獨立樣本，p-value 嚴重灌水，幾乎所有基因都「顯著」。對策：必須有生物重複，並用 pseudo-bulk。

The single most serious and most common scRNA-seq DE mistake — running FindMarkers directly on 1 disease sample vs 1 healthy sample. "2,000 vs 2,000 cells" actually means 1 vs 1 individual, but Wilcoxon treats them as 4,000 independent samples, severely inflating p-values — almost every gene becomes "significant." Mitigation: biological replicates + pseudo-bulk.

🚨 陷阱 2：小 cluster 的 p-value 不可靠

cluster 細胞數 < 10 時，無論用哪種方法都沒有統計檢力，且容易被 1-2 個 outlier 細胞主導。pseudo-bulk 中如果某 sample × cell_type 細胞數 < 10，DESeq2 也會給出失真結果。對策：刪除 < 10 細胞的 cluster 或合併到鄰近 cluster；pseudo-bulk 前過濾 min_cells = 10。

For clusters <10 cells, no method has power, and 1-2 outlier cells can dominate. In pseudo-bulk, any sample × cell-type combo with <10 cells also gives DESeq2 garbage. Mitigation: drop or merge tiny clusters; set min_cells = 10 when building pseudo-bulk.

🚨 陷阱 3：批次與條件完全混淆

若所有疾病樣本在 batch A 跑、所有對照在 batch B 跑，DE 結果無法區分「疾病效應」與「批次效應」——這是不可挽回的實驗設計缺陷。對策：實驗設計時務必混合（每個 batch 含 disease + control）；分析前用 table(sample, condition, batch) 檢查。若已混淆，須在論文中明確聲明 limitation。

If all disease samples run in batch A and all controls in batch B, DE results cannot disentangle "disease effect" from "batch effect" — an unfixable design flaw. Mitigation: at design time, ensure every batch contains both disease and control; check with table(sample, condition, batch) before analysis. If confounded, declare it as a limitation.

🚨 陷阱 4：環境 RNA 偽 marker

在肝臟、腦等富含高豐度基因（ALB、MBP）的組織，這些基因會「污染」所有液滴，導致它們在每個 cluster 都「表達」並以 marker 身分出現。對策：在 QC 時用 SoupX / CellBender 清理；若 marker 出現在多個本應無關的 cluster，警覺其為環境 RNA。

In tissues with abundant lineage genes (ALB in liver, MBP in brain), these genes contaminate all droplets, appearing "expressed" in every cluster and showing up as markers. Mitigation: clean with SoupX / CellBender during QC; if a "marker" appears across unrelated clusters, suspect ambient RNA.

🚨 陷阱 5：adj-p < 0.05 ≠ 生物學顯著

在 100,000 細胞的數據中，p-value 會非常小，但可能 logFC 只有 0.1（生物上幾乎沒差別）。對策：同時設 |log2FC| ≥ 0.5（甚至 1.0）+ pct.1 > 0.25。Volcano plot 上同時看 x（FC）與 y（-log10 p）兩軸。

With 100,000 cells, p-values can be vanishingly small even when logFC ≈ 0.1 (biologically irrelevant). Mitigation: require |log2FC| ≥ 0.5 (or 1.0) + pct.1 > 0.25. On volcano plots, evaluate both x (FC) and y (-log10 p).
