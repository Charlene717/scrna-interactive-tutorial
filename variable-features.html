STEP 3 / 9

高變異基因選擇

從數萬個基因中篩選出最具「分辨力」的特徵，聚焦生物學訊號、過濾噪音。

Select the most discriminative features from tens of thousands of genes — focus on biological signal, filter noise.

概覽

一、為什麼不用所有基因？

典型 scRNA-seq 數據包含 2–3 萬個基因，但其中 80% 以上對「區分細胞類型」沒有貢獻。它們可能是：(1) 管家基因（GAPDH、ACTB、HPRT1、TBP）——在所有細胞穩定表達；(2) 完全不表達的基因（嗅覺受體在血細胞、Hox 在成熟組織）；(3) 雜訊主導的低表達基因——平均表達 < 0.1，主要是測序錯誤與技術 dropout。

HVG 選擇做了什麼？在 mean-variance 空間中，找出「變異度超過深度與技術噪音預期」的基因。這些基因攜帶最強的細胞型別 / 狀態訊號，讓後續 PCA / clustering / UMAP 聚焦於有意義的差異，同時把運算量縮減 10 倍以上。

為什麼選 2000 個？經驗值：PBMC、骨髓等典型免疫圖譜常用 2000；複雜組織（腫瘤微環境、發育、神經）可調到 3000–5000；單一純化 lineage 1000 即可。Seurat 文章驗證 2000 在多數常見組合下能保留 > 95% 的下游分群結構。

A typical scRNA-seq dataset has 20,000–30,000 genes, but over 80% contribute nothing to distinguishing cell types. They fall into: (1) housekeeping genes (GAPDH, ACTB, HPRT1, TBP) — stably expressed everywhere; (2) fully off genes (olfactory receptors in blood, Hox in mature tissues); (3) noise-dominated low-expression genes — mean < 0.1, mostly sequencing errors and technical dropouts.

What does HVG selection do? In mean-variance space, it finds genes whose variability exceeds the noise expected from depth alone. These genes carry the strongest cell-type / state signal, focusing downstream PCA/clustering/UMAP on meaningful differences and reducing compute by 10× or more.

Why 2000? Empirically: PBMC, bone-marrow and typical immune atlases use 2000; complex tissues (tumor TME, development, neural) bump to 3000–5000; a single purified lineage works fine with 1000. The Seurat paper shows 2000 retains > 95% of downstream cluster structure across typical combinations.

💡
HVG ≠ 重要基因。HVG 篩出的是「方差大」基因，不一定具生物學重要性；可能包含應激基因、線粒體相關、核醣體等。實際 marker 篩選應在 DE 步驟結合差異表現倍數與生物意義。 HVG ≠ important genes. HVG selects "high-variance" genes, not necessarily biologically important ones — may include stress, mitochondrial, ribosomal genes. Real marker identification belongs to the DE step combining fold-change with biological meaning.
核心數學

二、核心數學

關鍵觀察：在計數型數據中，方差會隨平均值線性甚至超線性增長（Poisson 為 var = mean；負二項則 var = mean + mean²/θ）。如果直接按 variance 排序，將永遠是高表達基因贏出，但這只是「技術上方差大」不代表生物有意義。

解法 — Variance Stabilizing Transformation (VST)：

  1. 對所有基因計算 log10(mean) 與 log10(variance)。
  2. 用 local polynomial regression (loess) 擬合 log(var) ~ log(mean) 的趨勢線——這是「技術噪音預期」。
  3. 計算每個基因的標準化殘差：observed variance 與擬合曲線的差異，並除以擬合 SD。
  4. 排名取 top-N（預設 2000）作為 HVG。

這樣低表達基因若異常變異（罕見 marker）也能被選中；高表達但變異「正常」的基因（如 RPL18）則被排除。

standardized_variancei = Var(log10(xi)) / loess_fit(log10(meani))
HVG = top-N gene by descending standardized_variance

Key observation: In count data, variance scales with mean, often linearly or super-linearly (Poisson: var = mean; NB: var = mean + mean²/θ). Sorting by raw variance always favors high-expression genes — yet "high variance" here is just technical, not biological.

Solution — Variance Stabilizing Transformation (VST):

  1. For all genes, compute log10(mean) and log10(variance).
  2. Fit local polynomial regression (loess) for log(var) ~ log(mean) — this trend is the "expected technical noise".
  3. Compute each gene's standardized residual: observed variance minus fit, divided by fit SD.
  4. Rank and take top-N (default 2000) as HVG.

This way, low-expression genes with unusual variance (rare markers) can still be selected; high-expression but "normally variable" genes (RPL18) are filtered out.

standardized_variancei = Var(log10(xi)) / loess_fit(log10(meani))
HVG = top-N genes by descending standardized_variance

程式碼

五、實作範例

# ===== Seurat: VST 標準流程 =====
pbmc <- FindVariableFeatures(pbmc,
  selection.method = "vst",
  nfeatures = 2000)

# 看 top-10 HVG
top10 <- head(VariableFeatures(pbmc), 10)
print(top10)

# 視覺化 mean-variance plot
plot1 <- VariableFeaturePlot(pbmc)
plot2 <- LabelPoints(plot = plot1, points = top10, repel = TRUE)
plot2

# 排除特定基因（如 MT/RPS/RPL/HBB）
hvg <- VariableFeatures(pbmc)
hvg <- hvg[!grepl("^MT-|^RP[SL]|^HB[ABZ]", hvg)]
VariableFeatures(pbmc) <- hvg

# 若使用 SCTransform：HVG 自動完成
# pbmc <- SCTransform(pbmc, variable.features.n = 3000)
import scanpy as sc

# ===== Scanpy: seurat_v3 flavor 流程 =====
# 注意 seurat_v3 需要 raw counts → 在 normalize 之前呼叫
sc.pp.highly_variable_genes(adata,
    flavor="seurat_v3",
    n_top_genes=2000,
    batch_key=None)  # 多樣本時改成 'sample'

# 視覺化
sc.pl.highly_variable_genes(adata)

# 排除特定 gene 家族（黑名單）
blocklist = adata.var_names.str.startswith(("MT-", "RPS", "RPL", "HB"))
adata.var.loc[blocklist, "highly_variable"] = False

# 後續分析只用 HVG（可選；亦可保留全部僅在 PCA 用 HVG）
adata_hvg = adata[:, adata.var.highly_variable].copy()
# ===== 多樣本 / 批次感知 HVG (Seurat v5) =====
samples <- SplitObject(pbmc, split.by = "orig.ident")

# 每個樣本各自選 HVG
samples <- lapply(samples, function(x) {
  x <- NormalizeData(x)
  x <- FindVariableFeatures(x, nfeatures = 2000)
  x
})

# 跨樣本選共識 features（用於 IntegrateData）
shared_hvg <- SelectIntegrationFeatures(
  object.list = samples,
  nfeatures = 3000)

# shared_hvg 即為跨樣本最常出現的 HVG 排名
length(shared_hvg)  # 3000
最佳實務

六、最佳實務

🎯 N 該選多少？

2000 是 Seurat 預設；PBMC、骨髓等典型免疫常用。3000–5000 用於腫瘤微環境、發育、神經組織。1000 用於純化 lineage。Atlas-級 dataset 用 4000+。寧可多選少用，不要少選漏掉稀有亞群。

2000 is Seurat default — typical immune datasets (PBMC, BM). 3000–5000 for TME, development, neural. 1000 for purified lineage. Atlas-scale: 4000+. Better over-select than miss rare subpopulations.

🚫 HVG 黑名單

建議排除：MT-（^MT-/^mt-）、核醣體（^RP[SL]）、血紅蛋白（^HB[ABZ]）、性染色體（XIST/RPS4Y1）、應激（HSP/JUN/FOS）、細胞週期（依研究目的）。這些常因技術或干擾因素進入 HVG 卻無生物學焦點。

Suggested blocklists: MT- (^MT-/^mt-), ribosomal (^RP[SL]), hemoglobin (^HB[ABZ]), sex chromosome (XIST/RPS4Y1), stress (HSP/JUN/FOS), cell cycle (study-dependent). These often slip into HVG due to technical confounders but aren't the biological focus.

🧩 批次感知

多樣本時：Scanpy 用 batch_key='sample'，會在每樣本內排名後取共識；Seurat 用 SelectIntegrationFeatures()。否則 HVG 可能主要反映批次差異而非生物變異。

Multi-sample: Scanpy uses batch_key='sample' to rank within each sample first, then intersect; Seurat uses SelectIntegrationFeatures(). Otherwise HVG may reflect batch differences rather than biology.

🔄 整合後重選 HVG？

原則上不需要：Integration 用的就是初次 HVG 對齊。但若後續發現某個 cluster 不被既有 HVG 區分（DE 找不到 marker），可在該 subset 上重做局部 HVG。

Generally not needed: integration aligns the initial HVG. But if a cluster isn't separated by existing HVG (DE finds no markers), redo local HVG on that subset.

🔬 視覺化驗證

每次選完務必畫 VariableFeaturePlot() 並標記 top-10。檢查：(1) 是否覆蓋整個 mean 範圍（不只是高表達區）；(2) top-10 是否包含預期 marker（如 PBMC 看 MS4A1、CD8A、NKG7、LYZ）。

Always plot VariableFeaturePlot() and label top-10. Check: (1) does the selection span the full mean range (not just high-expression region)? (2) does top-10 include expected markers (e.g., PBMC: MS4A1, CD8A, NKG7, LYZ)?

♻️ HVG vs. SCT/Pearson

SCTransform 與 Pearson residuals 流程自動內建 HVG 選擇，不需呼叫 FindVariableFeatures。混用會造成困惑。新項目建議「擇一流程」並貫徹到 PCA。

SCTransform and Pearson residual workflows build in HVG selection — don't call FindVariableFeatures. Mixing causes confusion. Pick one workflow and follow it through PCA.

陷阱

七、常見陷阱

⚠️
HVG 常見的 4 個系統性失誤：
① HVG 由 1 個極端細胞主導——某基因僅在 1 個 doublet 高表達就上榜，後續會抓不到真實 marker。
② HVG 由批次差異驅動——多樣本未使用 batch-aware 選擇，整合後 cluster 全是 sample-driven。
③ HVG 太少漏掉稀有亞群——只取 500–1000 個會把罕見細胞（< 1% 比例）的 marker 直接排除。
④ 沒排除技術 / 雜訊基因——MT、RPS/RPL、HSPA1A 等常進入 top-100 但無生物意義。
Top 4 systematic HVG failures:
① One outlier cell drives HVG — a gene high only in a single doublet ranks high; downstream misses true markers.
② Batch effects drive HVG — multi-sample without batch-aware selection; clusters end up sample-driven after integration.
③ Too few HVG misses rare populations — N=500–1000 excludes markers of cells < 1% frequency.
④ Forgot to remove technical / noise genes — MT, RPS/RPL, HSPA1A often enter top-100 without biological focus.

🚨 Cluster 按樣本分群

原因：HVG 主要反映批次差異。
修正：Scanpy 加 batch_key；Seurat 改用 SelectIntegrationFeatures()；或先 Harmony / scVI 整合。

Cause: HVG reflects batch differences.
Fix: Scanpy add batch_key; Seurat use SelectIntegrationFeatures(); or integrate with Harmony/scVI first.

🚨 稀有細胞 marker 失蹤

原因：該 marker 未進入 HVG，PCA 看不到。
修正：N 從 2000 提高到 4000–5000；或對該 subset 重新 FindVariableFeatures + 子 PCA。

Cause: Marker isn't in HVG, so PCA doesn't see it.
Fix: Increase N from 2000 to 4000–5000; or re-run FindVariableFeatures + sub-PCA on the subset.

🚨 Top-10 都是 RPS/MT

原因：沒做 QC 或沒套用 blocklist。
修正：QC 收緊 percent.mt；或在 HVG 後手動移除 ^MT-|^RP[SL]|^HB。

Cause: QC too lenient or no blocklist.
Fix: Tighten percent.mt; manually remove ^MT-|^RP[SL]|^HB after HVG.

🚨 細胞週期主導

原因：樣本含高度增殖細胞（腫瘤、發育）；S/G2M 基因方差最大。
修正：若不研究細胞週期：ScaleData 階段 vars.to.regress=c('S.Score','G2M.Score')；或從 HVG 排除 Seurat::cc.genes 列表。

Cause: Sample contains highly proliferating cells (tumor, development); S/G2M genes have largest variance.
Fix: If not studying cell cycle: regress in ScaleData (vars.to.regress=c('S.Score','G2M.Score')); or exclude Seurat::cc.genes list from HVG.

🚨 合併後 HVG 不同

原因：合併數據的 mean-variance 結構受批次拉伸。
修正：使用 SelectIntegrationFeatures() 取每樣本 HVG 的共識排名。

Cause: Merged mean-variance is stretched by batch.
Fix: Use SelectIntegrationFeatures() to consensus-rank per-sample HVGs.

🚨 PCA 只看 HVG

提醒：HVG 之外的基因仍可用 FeaturePlot() 視覺化（讀 RNA assay），但不會影響 cluster 結構。「未進 HVG」≠「在資料中消失」。

Note: Non-HVG genes can still be visualized via FeaturePlot() (reads RNA assay) but don't affect clustering. "Not in HVG" ≠ "removed from data".
